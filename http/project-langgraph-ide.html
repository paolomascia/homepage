LangGraph IDE

.NET 8 WPF LangGraph Python
View on GitHub

LangGraph IDE is a standalone visual editor for designing LangGraph AI agent workflows. It lets you build complex multi-node agent graphs through an intuitive drag-and-drop interface and instantly exports them as ready-to-run Python code — no backend server required.

Why LangGraph IDE?

Designing multi-agent workflows with LangGraph typically means writing and mentally mapping out graph structures in code. LangGraph IDE flips this around: you design the graph visually, connect nodes intuitively, and the tool generates production-ready Python code in real time. It's aimed at AI engineers, researchers, and practitioners who want to prototype and iterate on agent architectures faster.

Key Features

  • Visual Graph Editor — Drag-and-drop canvas for placing nodes and drawing edge connections between them. The layout reflects the actual LangGraph structure.
  • Rich Node Library — A curated palette covering the full LangGraph ecosystem:
    • LLMs — OpenAI, Anthropic, Google Gemini, Mistral, Ollama
    • Tools — Tool-calling nodes for function integration
    • Memory — Short-term and long-term memory nodes
    • RAG — Retrieval-Augmented Generation components
    • Logic — Routing, decision, and conditional nodes
    • Custom — User-defined node types
  • Conditional Edges — Define routing logic directly on edges through an inline condition editor, without leaving the canvas.
  • Real-time Python Preview — A syntax-highlighted code panel (powered by AvalonEdit) updates live as you build the graph, showing exactly what Python code will be generated.
  • Export to Python — Export the complete agent graph as a .py file, ready to run with your LangGraph environment.
  • Save & Load — Persist and reload agent designs as .agent.json files for iterative development and sharing.
  • Dark Theme UI — A focused, distraction-free interface built entirely with WPF.
  • Keyboard Shortcuts — Full keyboard support for common operations: Ctrl+N (new), Ctrl+O (open), Ctrl+S (save), Ctrl+Z/Ctrl+Y (undo/redo), and more.

How It Works

  1. Design the Graph — Drop nodes from the library onto the canvas. Each node represents a component of your LangGraph pipeline: an LLM call, a tool invocation, a memory read, a RAG retrieval, or custom logic.
  2. Connect Nodes — Draw edges between nodes to define the flow. For conditional routing, open the inline edge editor to specify the routing condition directly.
  3. Preview the Code — The Python code panel on the right-hand side updates in real time as you modify the graph, giving immediate feedback on how the visual structure maps to LangGraph code.
  4. Export & Run — When the design is ready, export it as a .py file. The generated code is structured, readable, and ready to execute in any Python environment with LangGraph installed.

Technology Stack

  • .NET 8 — Runtime and application framework
  • WPF (Windows Presentation Foundation) — Desktop UI framework for the visual editor and canvas
  • AvalonEdit 6.3.1 — Syntax-highlighted, embeddable code editor for the Python preview panel
  • Newtonsoft.Json 13.0.3 — JSON serialization for saving and loading .agent.json graph files

Requirements

  • Windows 10 or Windows 11
  • .NET 8 Runtime
  • A Python environment with LangGraph installed (for running the exported code)

Node Categories

The node library is organized into eight categories to cover the full range of LangGraph agent patterns:

  • Control — Entry points, termination, and flow control nodes
  • Input / Output — Nodes for handling user input and agent output
  • LLM — Language model nodes for OpenAI, Anthropic, Gemini, Mistral, and Ollama
  • Tools — Function and API tool-calling nodes
  • Memory — Nodes for reading from and writing to agent memory
  • RAG — Retrieval-Augmented Generation nodes for document search and context injection
  • Logic — Conditional branching and routing decision nodes
  • Custom — User-defined nodes for extending the library

The project is open-source and available on GitHub.
