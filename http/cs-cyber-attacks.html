Cyber Attacks: Types, Techniques, and Threat Landscape

Cybersecurity Attack Vectors Threat Landscape
Educational Disclaimer
This document is for educational and defensive cybersecurity purposes only. The techniques described should only be used in authorized penetration testing environments. Unauthorized use of these techniques may be illegal and could result in criminal charges. Always obtain proper written authorization before testing any systems. Use this knowledge responsibly to improve security defenses.

The world of cybersecurity is vast and constantly evolving. Attackers use a wide variety of methods to compromise systems, disrupt operations, or gain unauthorized access to data. Understanding these attack types is essential for designing resilient systems, training users, and implementing layered defenses.

1) Denial of Service (DoS) Attacks

A Denial of Service (DoS) attack aims to make a system or service unavailable by overwhelming it with traffic or resource requests. When multiple compromised systems participate simultaneously, the attack becomes a Distributed Denial of Service (DDoS) attack.

Attackers flood target servers and networks with massive volumes of fake requests, exhausting bandwidth, CPU, or memory so that legitimate users cannot access services. Often, the attacking devices are themselves victims — infected machines in a botnet used without the owners’ consent.

Common types include:

  • TCP SYN Flood: Exploits the TCP handshake to consume resources by leaving half-open connections.
  • Ping of Death: Sends oversized or malformed ICMP packets causing system crashes.
  • Teardrop Attack: Sends fragmented packets that crash systems when reassembled incorrectly.
  • Smurf Attack: Uses ICMP echo requests to broadcast traffic from many systems toward a victim.

2) Malware

The term malware encompasses all types of malicious software designed to infiltrate, damage, or gain control over systems without user consent. Malware can be delivered through phishing emails, malicious downloads, infected USB drives, or vulnerabilities in web browsers and applications.

Common Malware Categories

  • Viruses: Self-replicating code that infects other files and spreads across systems.
  • Trojans: Malicious software disguised as legitimate applications, often used to create backdoors.
  • Worms: Self-propagating programs that spread through networks autonomously — commonly via email or file sharing.
  • Ransomware: Encrypts a victim’s files and demands payment (often in cryptocurrency) to restore access or prevent data leaks.
  • Spyware: Secretly collects user data, browsing habits, or credentials for misuse or sale.

Malware Delivery Methods

Malware often exploits human error or system flaws. Clicking on unsafe links, opening attachments, or visiting compromised websites can trigger infection. Some variants exploit browser or plugin vulnerabilities, executing code simply upon visiting a webpage (drive-by downloads).

Developers often provide checksums or digital signatures that allow users to verify authenticity before installation.

3) Man-in-the-Middle (MitM) Attacks

A Man-in-the-Middle (MitM) attack occurs when an attacker secretly intercepts and potentially alters communication between two parties. This allows the attacker to eavesdrop, steal credentials, or inject malicious content without either side’s awareness.

MitM attacks often target insecure Wi-Fi networks, unencrypted connections, or compromised routers. Although the widespread use of HTTPS and TLS has reduced risk, protocol exploits, weak certificates, and phishing techniques can still enable such attacks.

4) Password Attacks

Passwords remain one of the most common authentication mechanisms worldwide. Attackers often target weak or reused passwords using a range of methods:

  • Brute Force: Systematically tries every possible combination of characters until the correct password is found.
  • Dictionary Attack: Uses lists of common words or passwords (e.g., 123456, password, qwerty).
  • Credential Stuffing: Uses stolen username-password pairs from prior breaches to access new accounts.
  • Social Engineering: Exploits human psychology to trick users into revealing their passwords directly.

Complex passwords, password managers, and multi-factor authentication (MFA) greatly reduce exposure to these attacks.

5) SQL Injection (SQLi)

SQL Injection attacks target web applications that interact with databases. By injecting malicious SQL queries into input fields (e.g., login forms or comment sections), attackers can manipulate backend queries to access, modify, or delete sensitive information.

Poor input validation and unsafe database handling make systems vulnerable. Mitigation involves input sanitization, prepared statements, and least-privilege database permissions.

6) Cross-Site Scripting (XSS)

A Cross-Site Scripting (XSS) attack allows attackers to inject and execute malicious JavaScript in the browsers of other users. The script runs as if it were part of the trusted site, enabling credential theft, session hijacking, or redirection to phishing pages.

Preventing XSS requires strict input/output validation, context-aware encoding, and use of Content Security Policy (CSP).

7) Zero-Day Exploits

A Zero-Day exploit targets vulnerabilities unknown to software vendors or for which no patch exists. Attackers exploit these “day-zero” weaknesses before developers can respond. Such attacks are particularly dangerous because there are no existing defenses at the time of discovery.

Organizations must rely on proactive detection, behavioral analysis, and rapid patch management to mitigate the impact of zero-day threats.

8) Internet of Things (IoT) Attacks

The explosion of Internet of Things (IoT) devices — cameras, thermostats, sensors, and smart appliances — has created new attack surfaces. Many IoT devices lack robust security features, receive infrequent updates, and require open ports or insecure protocols for remote access.

Because these devices are connected to networks, they can be exploited as entry points or as components of large-scale botnets (e.g., Mirai). Manufacturers’ limited update cycles further compound risks, leaving vulnerabilities unpatched.

To defend IoT environments, organizations should isolate devices on separate network segments, disable unnecessary services, and change default credentials immediately.

Summary

The attack types described above represent only a fraction of today’s cybersecurity landscape. Every day, new vulnerabilities, tools, and threat actors emerge, increasing both the volume and sophistication of attacks. Effective defense requires continuous monitoring, timely patching, security awareness, and a layered security architecture that assumes compromise is possible.
