File Handling in Go (Golang): Read, Write, and Manage Files

Go File I/O Standard Library

File handling is one of the most common tasks in software development. Go’s standard library provides powerful and efficient tools for reading, writing, and managing files through the os, io, and bufio packages.

Creating and Writing Files

The simplest way to create and write to a file in Go is using os.Create(). This function creates a new file (or truncates an existing one) and returns a file handle you can write to.

package main

import (
    "fmt"
    "os"
)

func main() {
    file, err := os.Create("example.txt")
    if err != nil {
        fmt.Println("Error creating file:", err)
        return
    }
    defer file.Close()

    file.WriteString("Hello, Go File IO!\n")
    file.WriteString("This is a second line.\n")

    fmt.Println("File created and written successfully!")
}

Reading Entire Files

For small to medium files, os.ReadFile() loads the complete contents into memory as a byte slice. This is the simplest approach when you need the full content at once.

data, err := os.ReadFile("example.txt")
if err != nil {
    fmt.Println("Error reading file:", err)
    return
}
fmt.Println(string(data))

Line-by-Line Reading

For large files, reading line by line with bufio.Scanner is far more memory-efficient. This approach processes one line at a time without loading the entire file into memory.

package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    file, err := os.Open("example.txt")
    if err != nil {
        fmt.Println("Error opening file:", err)
        return
    }
    defer file.Close()

    scanner := bufio.NewScanner(file)
    for scanner.Scan() {
        fmt.Println(scanner.Text())
    }

    if err := scanner.Err(); err != nil {
        fmt.Println("Error reading file:", err)
    }
}

Appending to Files

To add content to an existing file without overwriting it, use os.OpenFile() with the os.O_APPEND flag. This opens the file in append mode so new writes go to the end.

file, err := os.OpenFile("example.txt", os.O_APPEND|os.O_WRONLY, 0644)
if err != nil {
    fmt.Println("Error opening file for append:", err)
    return
}
defer file.Close()

if _, err := file.WriteString("Appended line.\n"); err != nil {
    fmt.Println("Error writing to file:", err)
}

File Management

Go makes it straightforward to rename and delete files using functions from the os package.

os.Rename("example.txt", "renamed.txt")
os.Remove("renamed.txt")

File Permissions

You can control file permissions using os.Chmod(). Permissions are specified using Unix-style octal notation.

os.Chmod("example.txt", 0600) // owner read/write only

Binary Data Operations

For working with binary files, Go provides the encoding/binary package. This is useful when you need to read or write structured binary data such as integers, floats, or custom protocols.

package main

import (
    "encoding/binary"
    "fmt"
    "os"
)

func main() {
    file, _ := os.Create("numbers.bin")
    defer file.Close()

    numbers := []int32{10, 20, 30, 40}
    for _, n := range numbers {
        binary.Write(file, binary.LittleEndian, n)
    }

    fmt.Println("Binary file written.")
}

Best Practices

  • Handle errors properly — always check the error return value from file operations.
  • Use defer file.Close() immediately after opening a file to ensure it is always closed.
  • Prefer bufio.Scanner or bufio.Writer for large data to avoid loading everything into memory.
  • Check permissions and ensure cleanup of temporary files.
  • Handle relative vs. absolute paths carefully to avoid unexpected behaviour across environments.

Conclusion

Go’s standard library provides everything needed for powerful and efficient file handling. Whether you’re reading configuration files, processing logs, writing binary data, or managing file metadata, the os, io, and bufio packages give you a clean, idiomatic API to get the job done.
