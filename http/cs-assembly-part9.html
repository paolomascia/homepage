IA-32 and Intel 64 ISA Assembly Language (Part 9)

Assembly NASM Control Flow

High-level constructs such as for, if/else, and switch do not exist natively in assembly. Instead, developers compose them using comparisons (CMP), flag-sensitive conditional jumps (Jcc), and labels. This post expands each construct with fully annotated NASM examples (Linux x86 32-bit syscall exit shown), explains how the flags drive control flow, and highlights common pitfalls and 64-bit considerations.

Implementing a Loop

A classic count-down loop uses a register as the loop counter, tests against zero, executes the body, decrements, and jumps back. Below, ECX holds the loop count loaded from counter. The loop continues while ECX != 0.

section .data
    counter dd 10          ; initial loop count

section .text
    global _start

_start:
    mov  ecx, [counter]    ; ECX = remaining iterations

start_loop:
    cmp  ecx, 0            ; set ZF if ECX == 0
    je   end_loop           ; if zero, we're done

    ; --- loop body (work) ---
    ; ... your code using EAX/EBX/EDX/etc. ...
    ; avoid clobbering ECX unless reloaded

    dec  ecx               ; ECX = ECX - 1
    jmp  start_loop

end_loop:
    ; exit(0) via int 0x80 (Linux 32-bit)
    mov  eax, 1            ; sys_exit
    xor  ebx, ebx          ; status = 0
    int  0x80

CMP ecx, 0 subtracts 0 from ECX and sets flags. JE checks ZF. You can also use TEST ECX, ECX (faster on some CPUs) to set ZF/SF based on ECX without changing ECX.

Variations

  • Count-up: initialize ECX = 0, compare against a limit, INC ECX.
  • While(predicate): compute predicate flags first, jump to end if false.
  • Do/while: execute body once, then test at bottom before looping.

Implementing if–then–else

Compare a value and branch to the appropriate block. After executing the then part, jump over the else part.

section .text
    global _start

_start:
    mov  eax, 5            ; value under test
    cmp  eax, 5            ; sets ZF if EAX == 5
    jne  else_block        ; if not equal, go to else

then_block:
    ; ... code when (eax == 5) ...
    jmp  end_if

else_block:
    ; ... code when (eax != 5) ...

end_if:
    mov  eax, 1            ; sys_exit
    xor  ebx, ebx
    int  0x80

Flag choice matters: after CMP a, b, use signed jumps (JL/JG/...) when treating operands as signed, unsigned jumps (JB/JA/...) for size comparisons.

Implementing a switch Construct

A straightforward approach chains CMP/JE tests. For dense small ranges, a jump table is more efficient (shown after).

section .text
    global _start

_start:
    mov  eax, 2            ; switch (EAX)

    cmp  eax, 1
    je   case_1
    cmp  eax, 2
    je   case_2
    cmp  eax, 3
    je   case_3
    jmp  default_case

case_1:
    ; ... case 1 ...
    jmp  end_switch

case_2:
    ; ... case 2 ...
    jmp  end_switch

case_3:
    ; ... case 3 ...
    jmp  end_switch

default_case:
    ; ... default ...

end_switch:
    ; fall-through after switch

Jump Table (Dense Keys, Faster)

If cases are 0..N, use an index to jump via an address table. Guard out-of-range first.

section .rodata
jt:
    dd  case0, case1, case2, default_case  ; addresses (32-bit)

section .text
; assume EAX in 0..3
cmp  eax, 3
ja   default_case
jmp  dword [jt + eax*4]   ; indirect jump via table

case0: ; ...
case1: ; ...
case2: ; ...
default_case: ; ...

Arrays in Assembly

Arrays are contiguous memory blocks. Use db/dw/dd/dq based on element size. Compute addresses by base + index*elem_size + displacement.

section .data
    int_array  dd 10, 20, 30, 40, 50   ; 5 x 32-bit ints
    array_size equ 5

section .text
    global _start

_start:
    ; direct addressing by byte offset
    mov  eax, [int_array]        ; 1st (offset 0)
    mov  ebx, [int_array + 4]    ; 2nd (offset 4)
    mov  ecx, [int_array + 8]    ; 3rd (offset 8)

    ; store 99 into 3rd element
    mov  dword [int_array + 8], 99

    ; iterate by index
    xor  esi, esi                ; i = 0
    mov  edi, array_size
.loop:
    cmp  esi, edi
    jge  .done
    mov  eax, [int_array + esi*4] ; element i (4 bytes each)
    ; ... process EAX ...
    inc  esi
    jmp  .loop
.done:
    mov  eax, 1
    xor  ebx, ebx
    int  0x80

Element-size scaling factors are restricted to 1, 2, 4, or 8. For 64-bit code, prefer RIP-relative addressing: mov eax, [rel int_array + rsi*4].

Subroutine Parameter Passing (Stack)

On 32-bit Linux (cdecl), pass arguments right-to-left on the stack. The caller cleans the stack after CALL. Use a frame pointer (EBP) for easy parameter access at positive offsets.

section .text
    global _start

_start:
    ; call sum(10, 20)
    push dword 20            ; 2nd argument
    push dword 10            ; 1st argument
    call sum
    add  esp, 8              ; pop arguments

    ; exit(result)
    mov  ebx, eax            ; result in EAX
    mov  eax, 1
    int  0x80

; int sum(int a, int b)
sum:
    push ebp
    mov  ebp, esp
    mov  eax, [ebp + 8]      ; a
    add  eax, [ebp + 12]     ; a + b
    pop  ebp
    ret

64-bit Notes

  • System V AMD64 (Linux/BSD/macOS): args in RDI, RSI, RDX, RCX, R8, R9; return in RAX.
  • Windows x64: args in RCX, RDX, R8, R9, caller allocates 32-byte shadow space on stack; return in RAX.
  • Keep stack 16-byte aligned at each CALL.

How the Flags Drive Control Flow

Instruction Effect Typical Jcc
CMP a, b Sets flags as if computing a - b JE/JNE (ZF), JL/JG/JLE/JGE (signed: SF/OF/ZF), JB/JA/JBE/JAE (unsigned: CF/ZF)
TEST x, x AND without storing result JZ/JNZ depending on zero/non-zero

Building and Running (Linux Examples)

# Assemble to 32-bit object
nasm -f elf prog.asm -o prog.o
# Link for 32-bit ELF
ld -m elf_i386 -o prog prog.o
./prog

On 64-bit Linux, you can still assemble 32-bit but need multilib support. For native 64-bit, switch to -f elf64 and use syscall instead of int 0x80.

Common Pitfalls

  • Clobbering loop counters: if you must use ECX for other work, save/restore it around the body.
  • Signed vs unsigned jumps: choose the correct Jcc family for your data interpretation.
  • Stack alignment (x86-64): maintain 16-byte alignment prior to calls, and on Windows x64 reserve 32-byte shadow space.
  • Memory addressing: scale factors are restricted to 1, 2, 4, 8; choose correct element-size in load/store (e.g., dword for 32-bit ints).
