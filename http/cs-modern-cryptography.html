Modern Cryptography

Cryptography Symmetric Key Public Key

The modern era of cryptography begins with Claude Shannon, widely regarded as the father of mathematical cryptography. In 1949 he published Communication Theory of Secrecy Systems in the Bell System Technical Journal, followed shortly by the book Mathematical Theory of Communication with Warren Weaver. These works, together with Shannon’s broader contributions to information theory, established the theoretical foundations of both cryptography and cryptanalysis, clarifying what “secrecy” means and how to reason about it rigorously.

In the late 1960s and 1970s, public (non-classified) progress accelerated. IBM proposed the Data Encryption Standard (DES) as a practical block cipher for securing electronic communications in industry and finance. After review and modifications advised by the U.S. National Security Agency (NSA), DES was adopted by the National Bureau of Standards (now NIST) as a Federal Information Processing Standard (FIPS). It became the first widely available encryption algorithm endorsed by a national authority, sparking significant public and academic interest.

DES and its stronger variants (such as Triple DES, or 3DES) remained in wide use for decades. However, DES’s 56-bit key length proved too small against growing computational power and brute-force attacks. This led to the search for a replacement, culminating in the Advanced Encryption Standard (AES), selected by NIST in 2001 from the Rijndael algorithm designed by two Belgian cryptographers. Today, DES is considered insecure for new systems, while AES is the de-facto standard for modern symmetric encryption.

Asymmetric (Public-Key) Cryptography

A pivotal breakthrough arrived with Whitfield Diffie and Martin Hellman’s 1976 paper New Directions in Cryptography. They introduced a radically new approach to key establishment, now known as the Diffie–Hellman key exchange, and opened the door to a new class of algorithms: asymmetric (public-key) cryptography.

Before this, practical ciphers were symmetric-key: the same secret key was used for both encryption and decryption. The core challenge was key distribution: each pair of communicators needed to share a secret in advance, which does not scale and is operationally fragile when participants are many or keys must be rotated frequently.

In asymmetric cryptography, each user has a mathematically related key pair: a public key (which can be shared) and a private key (kept secret). Many but not all public-key systems have the property that knowing the public key does not feasibly reveal the private key with any method currently known. This enables anyone to encrypt to a recipient using the recipient’s public key, while only the recipient can decrypt using their private key. The same mathematical structure also enables digital signatures, where a private key produces a signature that anyone can verify with the corresponding public key.

Crucially, a single public/private key pair can serve many senders: possession of a recipient’s public key does not compromise security, and no per-pair shared secret is required. This surprising property made secure communication at Internet scale both possible and practical.

Definitions and Core Concepts

Cryptology, Cryptography, and Cryptanalysis

Cryptology is the overarching science of secure communication and consists of two branches: cryptography (the design and use of systems that protect information) and cryptanalysis (the study and practice of breaking or evaluating those systems). Cryptanalysis is essential to validate security claims; without it, cryptography would be untested theory.

Symmetric-Key Algorithms

Symmetric algorithms use the same (or trivially related) keys for encryption and decryption. They are typically fast, use comparatively small keys, and are best suited for encrypting large volumes of data (e.g., storage encryption, VPN tunnels). The main drawback is key distribution and management: all communicating parties must securely share the same secret key.

Asymmetric (Public-Key) Algorithms

Asymmetric or public-key algorithms use key pairs and rely on so-called one-way functions — mathematical problems easy to compute in one direction but infeasible to invert without special knowledge (the private key). Anyone can encrypt to a public key, but only the holder of the corresponding private key can decrypt. Public-key cryptography also underpins digital signatures, certificate infrastructures, and secure boot chains.

Cryptographic Protocols

Protocols specify how cryptographic algorithms are combined and used in the real world, handling negotiation, authentication, replay protection, and more. Transport Layer Security (TLS), used by every modern web browser, is a prime example: it orchestrates asymmetric key exchange, certificate validation, symmetric encryption, and message authentication to provide confidentiality and integrity for Internet traffic.

Hash Functions

A cryptographic hash function maps input of arbitrary length to a fixed-length output called a hash (or digest). Good hash functions are deterministic, collision-resistant, and preimage-resistant. In practice, hashes support integrity checks, digital signatures, password storage (with salts and slow schemes), and data deduplication. In protocols, message authentication often combines hashing with a secret (e.g., HMAC) to detect tampering.
