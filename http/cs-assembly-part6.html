IA-32 and Intel 64 ISA Assembly Language (Part 6)

Assembly IA-32 Addressing Modes

Part 6 provides a practical deep dive into x86 addressing modes and data storage allocation — covering register, immediate, and memory addressing in detail, along with NASM directives for defining and reserving data, the TIMES repetition directive, and the EQU constant mechanism.

Addressing Modes in x86 Assembly

Instructions process operands that specify where data originates or where it should be stored. Instructions may have no operands (like RET), one operand (like INC EAX), or typically two operands (like MOV EAX, EBX). The first operand serves as the destination while the second acts as the source, which typically remains unmodified.

Basic Addressing Modes

x86 supports three fundamental addressing mode categories:

  1. Register Addressing
  2. Immediate Addressing
  3. Memory Addressing

Register Addressing

In register addressing, operands are stored directly in CPU registers. The instruction can use a register as the source, the destination, or both. Register addressing provides the fastest performance since operands reside within the processor.

MOV EAX, COUNTER     ; Memory -> Register
MOV COUNTER, EAX     ; Register -> Memory
MOV EAX, EBX         ; Register -> Register

Immediate Addressing

This mode embeds constant literal values directly within instructions. The first operand can be a register or memory location, while the second must be a constant.

MOV EAX, 100         ; 100 -> EAX
ADD COUNTER, 1       ; COUNTER = COUNTER + 1

Numeric Constants

NASM permits expressing constants across multiple number systems:

Example Base Notes
200DecimalDefault format
0200DecimalLeading zero has no meaning
0d200 or 200dDecimalExplicit base
0xc8 or 0c8hHexadecimalPrefix 0x or suffix h
310q or 0q310OctalSuffix or prefix q
11001000b or 0b1100_1000BinarySuffix b or prefix 0b; underscores allowed

Memory Addressing

In memory addressing, one operand references a memory location. The address can be specified directly or calculated using registers.

Basic forms:

[number]                  ; Displacement only
[reg]                     ; Base only
[reg + reg*scale]         ; Base + (Index x Scale)
[reg + number]            ; Base + Displacement
[reg + reg*scale + number]; Base + Index x Scale + Displacement

Examples:

[750]                    ; Absolute displacement (address 0x2EE)
[rbp]                    ; Base register only
[rcx + rsi*4]            ; Base + index * 4
[rbp + rdx]              ; Implicit scale 1
[rbx - 8]                ; Negative displacement
[rax + rdi*8 + 500]      ; Full addressing form
[rbx + counter]          ; Using variable address as displacement

64-bit Mode Notes

  • All memory references are effectively 64-bit (using 64-bit registers)
  • Scale factors are limited to 1, 2, 4, or 8
  • Address calculation can use a base, an index, an optional scale, and an optional 32-bit displacement
  • RIP-relative addressing is available in 64-bit mode: [RIP + disp32]

Data Storage Allocation

Variables are declared in .data or .bss sections using define or reserve directives.

Syntax: [variable-name] define-directive initial-value [, initial-value, ...]

Each variable receives an offset within the data segment.

Define Directives

Directive Meaning Bytes Allocated
DBDefine Byte1
DWDefine Word2
DDDefine Doubleword4
DQDefine Quadword8
DTDefine Ten Bytes10

Examples:

choice      DB  'y'
number      DW  12345
neg_number  DW  -12345
big_number  DD  123456789
real_number1 DD 1.234
real_number2 DQ 123.456

Reserve Directives

Reserve directives allocate uninitialized memory, typically in .bss:

Directive Meaning Bytes Reserved
RESBReserve Bytes1 × count
RESWReserve Words2 × count
RESDReserve Doublewords4 × count
RESQReserve Quadwords8 × count
RESTReserve Ten Bytes10 × count

Examples:

section .bss
buffer      RESB 64         ; 64 bytes
array       RESD 100        ; 100 doublewords (400 bytes)
matrix      RESQ 16         ; 16 quadwords (128 bytes)

Multiple Initialization with TIMES

The TIMES directive repeats definitions a specified number of times:

spaces TIMES 10 DB ' '      ; Defines 10 space characters
zeros  TIMES 8  DD 0        ; Defines 8 doublewords of value 0

Constant Definitions (EQU Directive)

The EQU directive assigns names to constants or expressions at assembly time:

MAX_ATTEMPTS EQU 10
MOV ECX, MAX_ATTEMPTS

WIDTH  EQU 10
HEIGHT EQU 25
AREA   EQU WIDTH * HEIGHT

Constants defined with EQU occupy no memory; they are replaced by their values during assembly.

Summary

  • Addressing modes define how operands are accessed: via registers, constants, or memory.
  • Register and immediate modes are faster; memory addressing supports complex expressions with base, index, scale, and displacement.
  • NASM provides clear directives for defining and reserving data.
  • The TIMES directive simplifies repetitive initialization.
  • EQU defines constants, improving readability and maintainability.
