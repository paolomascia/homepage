IA-32 and Intel 64 ISA Assembly Language (Part 4)

Assembly IA-32 Data Types

Part 4 examines the data types supported by the IA-32 and Intel 64 architectures — from fundamental byte-level storage units and integer representations through floating-point formats and pointer types across different processor modes.

Data Types in IA-32 and Intel 64 Architectures

Fundamental Data Types

The x86 architecture defines a set of fundamental data units that serve as the building blocks for all higher-level data structures. These types are directly supported by the processor’s instruction set.

Data Type Size Description
Byte 8 bits Basic unit of storage; smallest directly addressable data type.
Word 16 bits Composed of two bytes; often used in 16-bit operations.
Doubleword (DWORD) 32 bits Composed of four bytes; standard unit in 32-bit mode.
Quadword (QWORD) 64 bits Composed of eight bytes; standard unit in 64-bit mode.
Double Quadword (DQWORD) 128 bits Composed of sixteen bytes; used in SIMD and vector operations.

These sizes correspond to the natural word size of the processor mode:

  • 16-bit processors operate on words.
  • 32-bit processors operate on doublewords.
  • 64-bit processors operate on quadwords.

Integer Data Types

The Intel 64 and IA-32 architectures support two major classes of integers:

  • Unsigned integers: Non-negative binary values.
  • Signed integers: Two’s complement binary values representing positive and negative numbers.

Unsigned Integers

Unsigned integers represent only non-negative values. The range depends on the number of bits available:

Type Range
Byte (8-bit)0 to 255
Word (16-bit)0 to 65,535
Doubleword (32-bit)0 to 4,294,967,295 (2³² − 1)
Quadword (64-bit)0 to 18,446,744,073,709,551,615 (2&sup6;&sup4; − 1)

Signed Integers

Signed integers use the two’s complement representation, allowing for both positive and negative values. The highest-order bit represents the sign (0 = positive, 1 = negative).

Type Bit Used for Sign Range
ByteBit 7−128 to +127
WordBit 15−32,768 to +32,767
DoublewordBit 31−2,147,483,648 to +2,147,483,647
QuadwordBit 63−9,223,372,036,854,775,808 to +9,223,372,036,854,775,807

Arithmetic operations automatically take sign into account, and flags in the EFLAGS/RFLAGS register (such as the OF and SF flags) are updated accordingly.

Floating-Point Data Types

Floating-point numbers represent real values using the IEEE 754 binary standard. The IA-32 architecture supports three floating-point formats through the x87 FPU and SIMD extensions:

Type Length Precision Normalized Range (Approximate)
Single Precision 32 bits 24 bits 1.18 × 10−38 to 3.40 × 1038
Double Precision 64 bits 53 bits 2.23 × 10−308 to 1.79 × 10308
Double Extended Precision 80 bits 64 bits 3.37 × 10−4932 to 1.18 × 104932

Extended precision format (80 bits) is unique to x87 FPU operations and provides additional accuracy beyond standard IEEE single and double precision formats. SSE and AVX SIMD extensions, in contrast, operate only on single and double precision values.

Pointers and Memory Addresses

A pointer is a value representing the memory address of an object or instruction. Pointer sizes and structures depend on the processor’s mode and memory model.

Pointer Types in Non-64-bit Modes

Pointer Type Composition Description
Near Pointer 16-bit or 32-bit offset References an address within the current segment. Common in flat memory models and most 32-bit applications.
Far Pointer 16-bit segment selector + 16- or 32-bit offset Used in segmented memory models where the segment must be explicitly identified.

Near pointers are typical in protected-mode programs running with a flat address space (where all segments overlap). Far pointers are mostly historical and were used extensively in 16-bit DOS and early Windows programming.

Pointer Types in 64-bit Mode

In 64-bit mode, the memory model is simplified:

  • A near pointer is a 64-bit effective address and represents any valid virtual address.
  • Far pointers still exist theoretically and can be 16:16, 16:32, or 16:64 (segment selector + offset) but are rarely used.

Segment registers (FS and GS) can still provide base offsets, but the flat address space model makes most addressing “near” in practice.

Summary

  • The architecture defines a hierarchy of data sizes from bytes (8-bit) to double quadwords (128-bit).
  • Integer types may be signed or unsigned, with two’s complement used for negative values.
  • Floating-point data types adhere to IEEE 754 single, double, and extended formats, providing a wide dynamic range.
  • Pointers represent memory locations, and their size and structure vary depending on mode (16-, 32-, or 64-bit).
  • Modern 64-bit systems operate using a flat memory model with 64-bit near pointers and minimal segmentation.
