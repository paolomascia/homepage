Object Oriented Programming in Go

Go OOP Standard Library

Go supports object-oriented programming but differs from classical OOP languages. The language intentionally omits traditional features such as inheritance and classes, favoring a simpler and more compositional design. Key OOP principles — encapsulation, composition, and polymorphism — are all supported in Go through its unique approach.

Encapsulation with Structs

Structs group related data fields under a single logical unit. Fields are either exported (capital letter) or unexported (lowercase):

type Person struct {
    Name string
    Age  int
}

type Employee struct {
    ID   int      // exported
    name string   // unexported
}

Struct instances can be created using literal syntax or the new() keyword:

p1 := Person{Name: "Alice", Age: 25}
p2 := new(Person)
p2.Name = "Bob"
p2.Age = 30

Methods and Receivers

Methods are functions associated with a type through a receiver parameter. Value receivers work on copies; pointer receivers operate on the original object:

// Method with value receiver
func (p Person) GetName() string {
    return p.Name
}

// Method with pointer receiver (allows modification)
func (p *Person) Birthday() {
    p.Age++
}

Example usage:

person := Person{Name: "Alice", Age: 25}
fmt.Println(person.GetName()) // "Alice"

person.Birthday()
fmt.Println(person.Age) // 26

Composition Instead of Inheritance

Go doesn’t support classical inheritance. Instead, it encourages composition by embedding types:

type Person struct {
    Name string
    Age  int
}

func (p Person) GetName() string {
    return p.Name
}

type Student struct {
    Person // embedded struct
    Grade  float32
}

Embedded fields and methods are automatically promoted:

student := Student{
    Person: Person{Name: "John", Age: 20},
    Grade:  85.5,
}

fmt.Println(student.GetName()) // "John" - method promoted from Person

Interfaces and Polymorphism

Interfaces define method signatures. Go uses implicit implementation — no explicit declaration needed:

type Shape interface {
    Area() float64
}

type Circle struct {
    Radius float64
}

func (c Circle) Area() float64 {
    return 3.14 * c.Radius * c.Radius
}

func printArea(s Shape) {
    fmt.Println("Area:", s.Area())
}

func main() {
    c := Circle{Radius: 5}
    printArea(c) // Circle satisfies Shape implicitly
}

Go Standard Library

The fmt Package

Provides formatted I/O functions:

package main

import "fmt"

func main() {
    // Print to console
    fmt.Println("Hello, World!")

    // Format string with variables
    name := "Mario"
    age := 30
    fmt.Printf("My name is %s and I am %d years old\n", name, age)

    // Read input from keyboard
    var input string
    fmt.Print("Enter a value: ")
    fmt.Scanln(&input)
    fmt.Println("You entered:", input)
}

The net/http Package

Complete HTTP client and server tools without external dependencies:

package main

import (
    "fmt"
    "net/http"
)

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hello from Go HTTP Server!")
}

func main() {
    http.HandleFunc("/", handler)
    fmt.Println("Server running on http://localhost:8080")
    http.ListenAndServe(":8080", nil)
}

The io/ioutil Package

File reading and writing utilities:

import (
    "io/ioutil"
    "log"
)

// Read file
data, err := ioutil.ReadFile("filename.txt")
if err != nil {
    log.Fatal(err)
}
fmt.Println(string(data))

// Write file
err = ioutil.WriteFile("output.txt", data, 0644)
if err != nil {
    log.Fatal(err)
}

The strconv Package

String and number conversions:

import "strconv"

// String to int
num, err := strconv.Atoi("123")

// Int to string
str := strconv.Itoa(123)

// String to float
f, err := strconv.ParseFloat("3.14", 64)

// Boolean parsing
b, err := strconv.ParseBool("true")

The encoding/json Package

JSON encoding and decoding:

import "encoding/json"

type Person struct {
    Name string `json:"name"`
    Age  int    `json:"age"`
}

// Encode to JSON
person := Person{Name: "John", Age: 30}
jsonData, err := json.Marshal(person)
if err != nil {
    log.Fatal(err)
}
fmt.Println(string(jsonData))

// Decode from JSON
var p Person
err = json.Unmarshal(jsonData, &p)
if err != nil {
    log.Fatal(err)
}
fmt.Println(p.Name, p.Age)

Summary

Go embraces OOP pragmatically through:

  • Structs for encapsulating data
  • Methods for attaching behavior to types
  • Interfaces for polymorphism and abstraction
  • Composition for reusability and flexibility

Combined with its rich standard library, Go provides tools for writing clean, efficient, and maintainable software with a modern, concurrent, and type-safe design.
