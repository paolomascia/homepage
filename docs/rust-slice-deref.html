Slices, Deref Coercions & Custom Deref

Rust Slices Deref

Slices are fat pointers: a pointer to a contiguous region of memory paired with a length. They are one of Rust's most versatile types, providing a unified interface for arrays, Vecs, strings, and any contiguous sequence without copying or allocating. Deref coercions are the mechanism that makes smart pointers and collection types behave transparently as their inner types, enabling APIs that accept &[T] to work with Vec<T> and arrays without conversion.

Slices in Depth

A slice &[T] is a view into a contiguous sequence of T values. It is a dynamically sized type (DST): its size is not known at compile time, only the size of the fat pointer (two words: pointer + length) is. You always use slices behind a reference or inside a smart pointer.

The key property: every collection that stores elements contiguously implements Deref<Target = [T]>, which means anywhere a &[T] is expected, a &Vec<T>, &Box<[T]>, or &[T; N] will be accepted via coercion. Writing functions that take &[T] instead of &Vec<T> makes them work with any of these sources.

Slice Creation and Operations

Creating slices from different sources, slicing ranges, split operations, and the full set of slice methods.

fn sum(data: &[i32]) -> i32 { data.iter().sum() }

fn main() {
    // From array
    let arr = [1, 2, 3, 4, 5];
    let slice: &[i32] = &arr;
    let partial: &[i32] = &arr[1..4]; // [2, 3, 4]

    // From Vec (via Deref)
    let v = vec![10, 20, 30, 40, 50];
    let vslice: &[i32] = &v;     // Vec<T> deref to [T]
    let mid: &[i32] = &v[1..3]; // [20, 30]

    // sum() accepts &[i32]: works with both
    println!("{}", sum(slice));  // 15
    println!("{}", sum(vslice)); // 150
    println!("{}", sum(&arr));   // 15 -- &[T; N] coerces to &[T]

    // Core slice methods
    let data = vec![3, 1, 4, 1, 5, 9, 2, 6];
    println!("len:   {}", data.len());
    println!("first: {:?}", data.first());      // Some(3)
    println!("last:  {:?}", data.last());       // Some(6)
    println!("get 2: {:?}", data.get(2));       // Some(4)
    println!("get 99:{:?}", data.get(99));      // None

    // Split operations -- all return pairs of slices, no allocation
    let (head, tail) = data.split_at(3);
    println!("head={head:?}, tail={tail:?}");

    let (init, last) = data.split_last().unwrap();
    println!("last={last}, rest={init:?}");

    // chunks / windows (revisited with slices)
    let chunks: Vec<&[i32]> = data.chunks(3).collect();
    println!("chunks: {chunks:?}");

    // contains and binary_search
    let sorted = vec![1, 2, 3, 4, 5];
    println!("contains 3: {}", sorted.contains(&3)); // true
    println!("binary_search 3: {:?}", sorted.binary_search(&3)); // Ok(2)

    // rotate_left / rotate_right
    let mut r = [1, 2, 3, 4, 5];
    r.rotate_left(2);
    println!("rotated: {r:?}"); // [3, 4, 5, 1, 2]

    // swap elements
    let mut s = [1, 2, 3];
    s.swap(0, 2);
    println!("swapped: {s:?}"); // [3, 2, 1]
}

Mutable Slices

Sorting, filling, copying between slices, and the split_at_mut trick for obtaining two non-overlapping mutable slices from one.

fn normalise(data: &mut [f64]) {
    let max = data.iter().cloned().fold(f64::NEG_INFINITY, f64::max);
    if max != 0.0 {
        for x in data.iter_mut() { *x /= max; }
    }
}

fn main() {
    let mut v = vec![3.0, 1.0, 4.0, 1.0, 5.0, 9.0, 2.0, 6.0];

    normalise(&mut v);
    println!("{:?}", v.iter().map(|x| format!("{x:.2}")).collect::<Vec<_>>());

    // fill: set all elements to a value
    let mut zeros = vec![0i32; 5];
    zeros.fill(42);
    println!("{zeros:?}"); // [42, 42, 42, 42, 42]

    // fill_with: set with a closure
    let mut counter = vec![0i32; 5];
    let mut n = 0;
    counter.fill_with(|| { n += 1; n });
    println!("{counter:?}"); // [1, 2, 3, 4, 5]

    // copy_from_slice: bulk copy between same-length slices
    let src = [10, 20, 30];
    let mut dst = [0i32; 3];
    dst.copy_from_slice(&src);
    println!("{dst:?}"); // [10, 20, 30]

    // split_at_mut: the only way to get two mutable slices from one
    let mut data = [1, 2, 3, 4, 5, 6];
    let (left, right) = data.split_at_mut(3);
    left[0]  = 99;
    right[0] = 77;
    println!("{data:?}"); // [99, 2, 3, 77, 5, 6]

    // Sorting with key
    let mut words = vec!["banana", "apple", "cherry", "fig", "date"];
    words.sort_unstable_by_key(|s| s.len());
    println!("{words:?}"); // ["fig", "date", "apple", "banana", "cherry"]
}

Slice Patterns in match

Rust's pattern matching works on slices, enabling clean structural decomposition of sequences. Slice patterns can match on fixed-length prefixes, suffixes, or use .. to ignore the middle portion.

Comprehensive slice pattern examples: empty, single element, fixed-length, prefix+rest, and nested patterns.

fn describe(data: &[i32]) -> String {
    match data {
        [] =>
            "empty".to_string(),
        [x] =>
            format!("single: {x}"),
        [first, second] =>
            format!("pair: {first} and {second}"),
        [first, .., last] =>
            format!("starts with {first}, ends with {last}, len={}", data.len()),
    }
}

fn sum_recursive(data: &[i32]) -> i32 {
    match data {
        [] => 0,
        [head, tail @ ..] => head + sum_recursive(tail),
    }
}

fn is_sorted(data: &[i32]) -> bool {
    match data {
        [] | [_] => true,
        [a, b, rest @ ..] => a <= b && is_sorted(&[&[*b], rest].concat()),
    }
}

fn main() {
    println!("{}", describe(&[]));              // empty
    println!("{}", describe(&[42]));            // single: 42
    println!("{}", describe(&[1, 2]));          // pair: 1 and 2
    println!("{}", describe(&[1, 2, 3, 4, 5])); // starts with 1, ends with 5, len=5

    println!("sum: {}", sum_recursive(&[1, 2, 3, 4, 5])); // 15

    println!("sorted [1,2,3]: {}", is_sorted(&[1, 2, 3])); // true
    println!("sorted [1,3,2]: {}", is_sorted(&[1, 3, 2])); // false

    // Pattern in function parameter (destructure directly)
    fn first_and_rest(data: &[i32]) -> Option<(i32, &[i32])> {
        if let [head, tail @ ..] = data {
            Some((*head, tail))
        } else { None }
    }
    println!("{:?}", first_and_rest(&[10, 20, 30])); // Some((10, [20, 30]))
}

Deref Coercions

Deref coercion is the compiler's automatic insertion of * (dereference) operations when a value of type &T is used where &U is expected and T: Deref<Target = U>. The compiler applies coercions repeatedly, walking the deref chain until it finds a type that matches. This is why you can pass &String where &str is expected, &Vec<T> where &[T] is expected, and &Box<T> where &T is expected.

Deref coercions apply in three contexts: when a function parameter type doesn't match the argument, when a method receiver doesn't match, and when the * operator is used on a smart pointer. They are entirely free at runtime: the compiler resolves them at compile time and emits the appropriate pointer arithmetic.

The full deref chain from &Box<String> to &str, and a function that benefits from accepting the most general type.

fn count_vowels(s: &str) -> usize {
    s.chars().filter(|c| "aeiouAEIOU".contains(*c)).count()
}

fn process_slice(data: &[i32]) -> i32 {
    data.iter().sum()
}

fn main() {
    // &String -> &str (String: Deref<Target=str>)
    let s = String::from("hello world");
    println!("{}", count_vowels(&s));      // works

    // &Box<String> -> &String -> &str (two coercions)
    let boxed = Box::new(String::from("hello"));
    println!("{}", count_vowels(&boxed)); // works

    // &Vec<T> -> &[T] (Vec: Deref<Target=[T]>)
    let v = vec![1, 2, 3, 4, 5];
    println!("{}", process_slice(&v));     // works

    // &[T; N] -> &[T] (arrays coerce to slices)
    let arr = [10, 20, 30];
    println!("{}", process_slice(&arr));   // works

    // &Box<Vec<i32>> -> &Vec<i32> -> &[i32]
    let bv = Box::new(vec![1, 2, 3]);
    println!("{}", process_slice(&bv));    // works

    // Method calls also trigger deref coercion
    let boxed_str = Box::new(String::from("hello"));
    println!("{}", boxed_str.to_uppercase()); // Box -> String -> str method

    // Deref chain summary:
    // &Box<T>    -> &T
    // &Rc<T>     -> &T
    // &Arc<T>    -> &T
    // &Vec<T>    -> &[T]
    // &String    -> &str
    // &PathBuf   -> &Path
    // &OsString  -> &OsStr
}

Implementing Custom Deref

Implementing Deref for your own types allows them to participate in deref coercions and enables the * operator. This is most useful for newtype wrappers (thin wrappers around another type) and smart pointer types. The convention is to implement Deref only when your type genuinely behaves like a smart pointer to the target type — not as a shortcut for accessing a field.

A Buffer<T> newtype wrapping Vec<T> with custom methods, implementing Deref and DerefMut to inherit all [T] methods transparently.

use std::ops::{Deref, DerefMut};
use std::fmt;

/// A fixed-capacity buffer that wraps Vec<T> and adds domain-specific methods.
struct Buffer<T> {
    data:     Vec<T>,
    capacity: usize,
}

impl<T> Buffer<T> {
    fn new(capacity: usize) -> Self {
        Buffer { data: Vec::with_capacity(capacity), capacity }
    }

    fn push(&mut self, item: T) -> bool {
        if self.data.len() < self.capacity {
            self.data.push(item);
            true
        } else {
            false
        }
    }

    fn is_full(&self)  -> bool { self.data.len() == self.capacity }
    fn remaining(&self) -> usize { self.capacity - self.data.len() }
}

// Deref<Target=[T]> gives Buffer all [T] methods: iter, sort, windows, etc.
impl<T> Deref for Buffer<T> {
    type Target = [T];
    fn deref(&self) -> &[T] { &self.data }
}

impl<T> DerefMut for Buffer<T> {
    fn deref_mut(&mut self) -> &mut [T] { &mut self.data }
}

impl<T: fmt::Debug> fmt::Display for Buffer<T> {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "Buffer({}/{}) {:?}", self.data.len(), self.capacity, self.data)
    }
}

fn print_slice(s: &[i32]) { println!("slice: {s:?}"); }

fn main() {
    let mut buf: Buffer<i32> = Buffer::new(5);
    for i in 1..=6 { println!("push {i}: {}", buf.push(i)); }
    // push 1..5: true; push 6: false (full)

    println!("{buf}"); // Buffer(5/5) [1, 2, 3, 4, 5]

    // Deref coercion: &Buffer<i32> -> &[i32]
    print_slice(&buf); // works!

    // All [T] methods available via Deref
    println!("sum: {}", buf.iter().sum::<i32>()); // 15
    println!("first: {:?}", buf.first()); // Some(1)
    println!("contains 3: {}", buf.contains(&3)); // true

    // DerefMut: mutable [T] methods
    buf.sort_unstable_by(|a, b| b.cmp(a)); // sort descending via [T]::sort_unstable_by
    println!("sorted desc: {:?}", &buf[..]); // [5, 4, 3, 2, 1]

    println!("remaining: {}", buf.remaining()); // 0
    println!("is_full:   {}", buf.is_full());   // true
}

Box<[T]> — The Boxed Slice

A Vec<T> stores three fields: pointer, length, and capacity. When you know the size will never change, you can convert to a boxed slice Box<[T]>, which stores only a pointer and a length. This is more memory-efficient and communicates intent: the data is fixed in size. Converting is free (no reallocation if the Vec's length equals its capacity).

Converting Vec to boxed slice, using it as a fixed-size buffer, and the memory layout comparison.

fn main() {
    // Build with Vec, then freeze into a boxed slice
    let v: Vec<i32> = (1..=5).collect();
    let boxed: Box<[i32]> = v.into_boxed_slice(); // no reallocation if len == cap

    // Box<[T]> derefs to [T], so all slice methods work
    println!("len: {}", boxed.len());
    println!("sum: {}", boxed.iter().sum::<i32>()); // 15

    // Cannot push -- it's fixed size
    // boxed.push(6); // compile error: no `push` method on [i32]

    // Convert back to Vec
    let v2: Vec<i32> = boxed.into_vec();
    println!("{v2:?}");

    // Fixed-size buffer from a closure
    let squares: Box<[u32]> = (1..=10).map(|x: u32| x * x).collect();
    println!("{squares:?}");

    // Memory sizes
    println!("Vec<i32> size: {} bytes", std::mem::size_of::<Vec<i32>>());    // 24 (ptr+len+cap)
    println!("Box<[i32]> size: {} bytes", std::mem::size_of::<Box<[i32]>>()); // 16 (ptr+len)
    println!("&[i32] size: {} bytes",    std::mem::size_of::<&[i32]>());       // 16 (ptr+len)
}
