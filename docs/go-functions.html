The Functions

Go Functions Structs

Functions are the building blocks of any Go program. From simple procedures to methods on types, Go offers a clean and expressive function model — including multiple return values, pointer-based parameter passing, structs, interfaces, and a powerful package system.

Function Declaration

Function without Return Value

func functionName(parameter1 type1, parameter2 type2, ...) {
    // function body
}

func add(a, b int) {
    fmt.Println(a + b)
}

Function with Return Value

func functionName(parameter1 type1, parameter2 type2, ...) returnType {
    // function body
    return value
}

func add(a, b int) int {
    return a + b
}

Multiple Return Values

func swap(a, b int) (int, int) {
    return b, a
}

Methods on Types

type Rectangle struct {
    width  float64
    height float64
}

func (r Rectangle) Area() float64 {
    return r.width * r.height
}

Parameter Passing

Functions receive arguments through value passing, meaning copies are transmitted rather than original references. Pointers enable modification of original values:

func double(x *int) {
    *x *= 2
}

func main() {
    x := 42
    double(&x)
    fmt.Println(x) // Prints 84
}

Function Visibility

Public Functions

Functions with an uppercase first letter are exported and accessible from other packages:

package mypackage

func MyFunction() {
    // do something
}

Private Functions

Functions with a lowercase first letter are unexported and only accessible within the same package:

package mypackage

func myFunction() {
    // do something
}

Structs

Structs group related data fields into a single composite type. They can be nested and accessed via dot notation or through pointers:

type Person struct {
    Name    string
    Age     int
    Address Address
}

type Address struct {
    Street  string
    City    string
    State   string
    Zipcode string
}

// Creating and using structs
person := Person{
    Name: "Mario",
    Age:  30,
    Address: Address{
        Street:  "123 Main St",
        City:    "Milan",
        State:   "MI",
        Zipcode: "12345",
    },
}

// Accessing fields
fmt.Println(person.Name) // Prints "Mario"

// Using pointers to structs
personPtr := &person
fmt.Println(personPtr.Age) // Prints 30

Interfaces

Interfaces define a set of method signatures. Any type that implements all methods of an interface implicitly satisfies it:

type Shape interface {
    Area() float64
    Perimeter() float64
}

type Rectangle struct {
    Width  float64
    Height float64
}

func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

// Using the interface
var s Shape = Rectangle{Width: 10, Height: 20}
fmt.Println(s.Area())      // Prints 200
fmt.Println(s.Perimeter()) // Prints 60

Packages

Packages organize Go code into reusable units. The import statement brings external packages into scope:

import "fmt"

func main() {
    fmt.Println("Hello, world!")
}

Go modules allow you to easily and organizedly manage project package dependencies. The go.mod file contains dependency information and go.sum stores checksums for verification.
