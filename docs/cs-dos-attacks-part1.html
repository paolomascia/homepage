DoS Attacks (Part 1): Understanding the Basics

Cybersecurity DoS/DDoS Network Security
Educational Disclaimer
This document is for educational and defensive cybersecurity purposes only. The techniques described should only be used in authorized penetration-testing environments. Unauthorized use of these techniques may be illegal and could result in criminal charges. Always obtain proper written authorization before testing any systems. Use this knowledge responsibly to strengthen cybersecurity defenses.

A Denial-of-Service (DoS) attack aims to interrupt or degrade the normal operation of a server or network service by overwhelming it with fake or redundant traffic. When this attack is launched from multiple compromised systems, it becomes a Distributed Denial-of-Service (DDoS) attack.

Attackers often infect machines with malware, turning them into remotely controlled bots or zombies. A coordinated network of such devices, called a botnet, can send enormous amounts of requests toward a single target, making it extremely difficult for the server to distinguish malicious traffic from legitimate users.

Classification of DoS and DDoS Attacks

These attacks can generally be divided into three main categories:

1) Volume-Based Attacks

These include UDP floods, ICMP floods, and other bandwidth-saturation techniques. The goal is to exhaust the target’s network capacity, and attack strength is measured in bits per second (bps).

2) Protocol Attacks

Protocol-level attacks, such as SYN floods, Ping of Death, and Smurf attacks, consume the actual server resources or those of intermediary devices like firewalls or load balancers. They are measured in packets per second (pps).

3) Application-Layer Attacks

These target vulnerabilities at the web-application level (Layer 7). Examples include HTTP floods, Slowloris, and RUDY (“R-U-Dead-Yet”) attacks. Because they mimic normal user requests, they are difficult to detect and are measured in requests per second (rps).

Common DoS Attack Types by OSI Layer

OSI Layer Attack Category Examples
Application (Layer 7) Application Layer HTTP Flood, Slowloris
Transport (Layer 4) Protocol SYN Flood, UDP Flood
Network (Layer 3) Volume-Based ICMP Flood, IP Fragment

The Slowloris Attack Explained

The Slowloris attack demonstrates how even a single system can disrupt a web server without massive bandwidth. It works by sending many incomplete HTTP requests and keeping each connection open as long as possible.

A standard HTTP request has three parts:

1. Start Line: defines the HTTP method and target resource

GET /index.html HTTP/1.1

2. Headers: key–value pairs providing metadata

User-Agent: ExampleBrowser/1.0
Content-Type: text/html; charset=utf-8

3. Body (optional): data sent with POST requests (e.g., form fields)

In a Slowloris scenario, the attacker sends only the start line and headers very slowly — delaying each line — so the server keeps the connection open, waiting for the request to complete. With enough open half-complete connections, the server eventually runs out of available sockets and cannot respond to legitimate clients.

Defense Techniques

  • Set strict timeouts for incomplete HTTP headers.
  • Limit simultaneous connections per IP address.
  • Use reverse proxies, load balancers, or web application firewalls (WAFs).
  • Employ rate-limiting and behavior-based detection to identify slow or abnormal connections.

The RUDY Attack (“R-U-Dead-Yet”)

The RUDY attack is conceptually similar to Slowloris but targets the HTTP body instead of the headers. It opens multiple POST requests and transmits the request body extremely slowly, sending just a few bytes at a time.

By keeping each connection alive for a prolonged period, RUDY consumes web server threads and resources until legitimate traffic can no longer be processed.

Defensive Countermeasures

  • Implement request-timeout limits for incomplete POST data.
  • Use anomaly detection to identify clients that send data unusually slowly.
  • Offload HTTP parsing to specialized reverse-proxy servers.
  • Enable connection limits per source and enforce request-rate controls.

Summary

DoS and DDoS attacks exploit resource limitations — whether bandwidth, processing power, or connection capacity. Understanding how these attacks work at different layers of the OSI model is critical for building resilient, fault-tolerant systems.

In upcoming sections, we will analyze more advanced DoS mitigation strategies and explore how content delivery networks (CDNs), rate-limiting, and AI-driven detection mechanisms can help neutralize such attacks before they cause disruption.
