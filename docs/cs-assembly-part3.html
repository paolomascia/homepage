IA-32 and Intel 64 ISA Assembly Language (Part 3)

Assembly IA-32 Addressing

Part 3 of this series dives deep into operand and addressing modes on x86 processors, covering operand size attributes, immediate and register operands, memory addressing with base/index/scale/displacement components, and the critical differences between legacy and 64-bit addressing.

Operand and Address Size Attributes

In protected mode, each code segment carries a default operand size and address size determined by the segment’s D flag:

  • D = 1: default 32-bit operands and 32-bit addresses
  • D = 0: default 16-bit operands and 16-bit addresses

In real mode, virtual-8086 mode, and SMM, the default operand and address sizes are 16-bit. Prefixes can override the defaults on a per-instruction basis.

Mode D Flag Operand Size Prefix Address Size Prefix Effective Operand Size Effective Address Size
16-bit0NoNo1616
16-bit0YesNo3216
32-bit1NoNo3232
32-bit1YesNo1632

Override Prefix Bytes (Encodings)

  • Operand-size override: 66h (toggles between 16↔32 in legacy modes)
  • Address-size override: 67h (toggles between 16↔32 addressing in legacy modes)
  • Segment override: 2Eh 36h 3Eh 26h 64h 65h for CS, SS, DS, ES, FS, GS

64-bit Mode Specifics

  • Default operand size: 32-bit. Changeable to 64-bit by REX.W = 1 prefix. The 66h prefix selects 16-bit operands (REX.W, if present, overrides 66h)
  • Default address size: 64-bit. The 67h prefix selects 32-bit addressing (16-bit addressing generally unsupported in 64-bit mode)
  • Write-zero extension rule: writing to a 32-bit GPR (e.g., EAX) zero-extends into the full 64-bit register (RAX)
  • RIP-relative addressing: memory operands can be addressed relative to the current RIP (useful for position-independent code)

Operand Addressing

IA-32/Intel 64 instructions can use zero or more operands. Some are implicit (e.g., MUL uses EAX/RAX) and some explicit. A source operand may reside in:

  • The instruction itself (immediate)
  • A register
  • A memory location
  • An I/O port (with IN/OUT instructions)

Immediate Operands

Immediates are encoded directly in the instruction. Size follows the instruction’s effective operand size unless a special form is defined.

; Adds the immediate 14 to EAX (32-bit add)
ADD EAX, 14
  • Some 64-bit forms use a sign-extended imm32 to load 64-bit registers (e.g., MOV r64, imm32 sign-extends to 64 bits)
  • Encoding may include imm8 with implicit sign-extension for certain ALU ops (saves code size)

Register Operands

Depending on the instruction, register operands can be:

  • General-purpose: 8/16/32/64-bit GPRs (e.g., AL/AX/EAX/RAX, …, R8–R15 in 64-bit mode)
  • Segment registers: CS, DS, SS, ES, FS, GS (limited instruction set)
  • EFLAGS/RFLAGS (via dedicated instructions: PUSHF/POPF, LAHF/SAHF subset, etc.)
  • x87 FPU: ST0–ST7
  • MMX: MM0–MM7
  • XMM/YMM/ZMM and MXCSR for SIMD and FP control

Convention matters: for example, ECX/RCX is the loop counter for REP string ops; RDI/RSI are default destination/source for string moves in 64-bit System V ABIs during REP MOVSx.

Memory Operands

Memory operands are formed from a segment selector (implicit or overridden) and an effective address (EA). In legacy modes you often see “16:16” or “16:32” notation (segment:offset). In 64-bit mode, segmentation is largely disabled for addressing; FS and GS can still provide a base via segment overrides.

Offsets (effective addresses) are built from the components below:

  • Displacement (disp8/disp16/disp32)
  • Base (any GPR except RSP has special encoding rules; RBP/EBP as base implies disp if needed)
  • Index (any GPR except RSP in SIB; in 64-bit, R12 acts like RSP in encoding)
  • Scale (1, 2, 4, 8 multiplies the index)

Common EA Forms (Intel Syntax)

[number]                     ; displacement only
[reg]                        ; base only
[reg + reg*scale]            ; index*scale (+ optional base if encoding allows)
[reg + number]               ; base + displacement
[reg + reg*scale + number]   ; full form: base + index*scale + displacement
FS:[reg + disp]              ; with explicit segment override (legacy & 64-bit)
[RIP + disp32]               ; RIP-relative (64-bit only), disp is signed 32-bit

64-bit Addressing Notes

  • Default address size is 64-bit; 67h selects 32-bit addressing for that instruction
  • RIP-relative uses a signed 32-bit displacement added to the address of the next instruction; useful for position-independent code and data
  • FS/GS base can be used for thread-local storage or OS data. Segment prefixes 64h/65h still apply

Putting It Together: Effective Sizes and Examples

Operand vs Address Size

An instruction has two independent attributes:

  • Operand size: width of the data being operated on (affects immediate width, register width, and how memory data is read/written)
  • Address size: width used to compute effective addresses (size of base/index and displacement interpretation)
; Legacy 32-bit default (D=1): 32-bit operands, 32-bit addressing
ADD EAX, DWORD PTR [EBX + ECX*4 + 8]

; Operand-size override (66h) toggles to 16-bit operand, address remains 32-bit
ADD AX, WORD PTR [EBX + ECX*4 + 8]

; Address-size override (67h) toggles to 16-bit addressing in 16-bit segments,
; or 32-bit addressing in 64-bit mode
MOV EAX, DWORD PTR [EBX]        ; default addr size (32 in legacy, 64 in x64)
MOV EAX, DWORD PTR [ECX]        ; with 67h in x64, use 32-bit addressing

; 64-bit mode: default operand = 32, address = 64
ADD EAX, DWORD PTR [RDI + RSI*2 + 16]

; 64-bit operand via REX.W
REX.W + ADD RAX, QWORD PTR [RDI + RSI*2 + 16]

; 64-bit mode: write to 32-bit reg zero-extends
MOV EAX, 1                      ; RAX becomes 0x00000000_00000001

Immediates and Sign-Extension

; 64-bit move with sign-extended imm32
MOV RAX, 0xFFFFFFFF             ; encodes as imm32, becomes -1 sign-extended
MOV RAX, 0x00000001             ; encodes as imm32, becomes +1 sign-extended

; Using 66h in 64-bit mode selects 16-bit operand (when valid)
ADD AX, 5

Segment Registers and Overrides

By default the segment used for address calculation is determined by the addressing form (e.g., SS for stack addressing with EBP/RBP/ESP/RSP, otherwise DS). You may override with segment prefixes:

MOV EAX, FS:[ECX + 8]    ; force FS segment
MOV EAX, GS:[RCX + 8]    ; 64-bit mode, GS base used

In 64-bit mode, CS, DS, ES, and SS are largely ignored for linear addressing; FS and GS retain special roles.

Quick Reference

  • Use 66h to switch operand size (16↔32 in legacy; 16 in x64 unless REX.W set)
  • Use 67h to switch address size (16↔32 in legacy; 64↔32 in x64)
  • Use REX.W=1 to get 64-bit operands in 64-bit mode
  • In 64-bit mode, writing a 32-bit GPR zero-extends to 64 bits
  • Prefer RIP-relative addressing for position-independent data/code on x64
