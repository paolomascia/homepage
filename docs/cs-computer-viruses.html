Computer Viruses: How They Work, How They Spread, and How to Defend

Cybersecurity Malware Defense

A computer virus is malicious software that replicates by inserting a copy of itself into another executable program. Unlike standalone malware, viruses require a host to execute and spread. This article examines how viruses propagate, the various classification types, code insertion techniques, and a comprehensive defensive framework for prevention, detection, and response.

Disclaimer: This content serves educational and defensive cybersecurity purposes only. Do not create, modify, or deploy malware. Only perform security testing with explicit authorization.

What Is a Computer Virus?

A computer virus is malicious software that replicates by inserting a copy of itself into another executable program. Viruses require a host program to execute and spread — unlike worms, which can propagate independently across networks. Once activated, a virus can corrupt data, steal information, disrupt operations, or serve as an entry point for further attacks.

Propagation Methods

Viruses spread through five primary distribution channels:

  1. Email and messaging attachments: Malicious files disguised as legitimate documents, invoices, or updates.
  2. Drive-by downloads and malvertising: Compromised or malicious websites that exploit browser vulnerabilities to deliver payloads without user interaction.
  3. Untrusted freeware/installers: Bundled malware within seemingly legitimate software downloads from unofficial sources.
  4. Social media links and shareable content: Deceptive links spread through social platforms that redirect to malicious payloads.
  5. Removable media and network shares: Infected USB drives, external disks, or shared network folders that spread the virus when accessed.

Infection Symptoms

Common indicators of a virus infection include:

  • Suspicious pop-ups and unexpected dialog boxes
  • Unauthorized browser setting changes (homepage, search engine)
  • Mass emails sent from compromised accounts without user knowledge
  • Frequent application and system crashes
  • Significant system slowdowns and high resource usage
  • Unknown programs appearing in startup configuration

Virus Classification

Viruses are categorized by their behavior, target, and propagation technique:

  • Boot sector virus: Infects the system boot code, loading before the operating system and gaining early execution control.
  • Web scripting virus: Exploits browser vulnerabilities through malicious scripts embedded in web pages.
  • Browser hijacker: Alters browser settings and redirects traffic to malicious or ad-laden websites.
  • Resident virus: Loads into memory during runtime, persisting even after the infected program closes.
  • Direct-action virus: Activates only when the infected file executes, then immediately seeks new targets.
  • Polymorphic virus: Mutates its code with each infection cycle to evade signature-based detection.
  • File infector: Targets executables and operating system files, embedding malicious code within them.
  • Multipartite virus: Uses multiple infection vectors simultaneously, combining boot sector and file infection techniques.
  • Macro virus: Embedded in document macros (Word, Excel), executing when the document is opened with macros enabled.

Code Insertion Techniques

Viruses employ four primary strategies to embed themselves within host files:

1. File Beginning with Destructive Overwriting

The virus replaces the initial sections of the host file with its own code. This technique is easily detected because the infected program immediately malfunctions, alerting users and security tools.

2. Random Position with Overwriting

The virus inserts its code at random offsets within the file. This can cause system crashes if critical code sections are corrupted, making the infected file unstable and eventually detectable.

3. Appended Viruses

The virus attaches its malicious code to the end of the host file and redirects the entry point to execute the virus first. The original program still appears to function normally, making this technique more stealthy than overwriting approaches.

4. Cavity (Gap) Viruses

The most stealthy technique in this set: the virus locates unused padding or cavities inside the file and fills them with malicious code. The file size and apparent functionality remain largely unchanged, making detection significantly more difficult.

Defensive Framework

Prevention

  • Patch management: Regularly update operating systems, applications, and firmware to close known vulnerabilities.
  • Application control: Whitelist approved applications and block unauthorized executables.
  • Email/web filtering with sandboxing: Inspect attachments and downloads in isolated environments before delivery.
  • Least privilege access: Restrict user permissions to minimize the impact of a compromised account.
  • User security training: Educate users on phishing, social engineering, and safe computing practices.
  • Regular offline backups: Maintain backups disconnected from the network to ensure recovery capability.

Detection

  • Endpoint protection and XDR: Deploy modern endpoint detection and response solutions with behavioral analysis.
  • Macro controls: Disable macros from untrusted sources and require signed macros for Office documents.
  • File integrity monitoring: Track changes to critical system files and executables.
  • Network analytics: Monitor for anomalous traffic patterns indicating data exfiltration or C2 communication.

Response

  • Network isolation: Immediately isolate affected systems to prevent lateral spread.
  • Malware eradication and reimaging: Remove the infection and rebuild compromised systems from known-good images.
  • Credential rotation: Reset passwords and invalidate sessions for potentially compromised accounts.
  • Integrity validation: Verify system and data integrity post-eradication.
  • Post-incident analysis: Conduct root cause analysis and improve controls based on lessons learned.

Conceptual Policy Examples

Email Gateway Policy

# Conceptual email gateway policy (non-functional)
if attachment_type in {exe, js, vbs, ps1, macro-enabled} and sender not trusted:
    hold_for_sandbox()
    warn_recipient()

Office Macro Policy

  • Disable macros from Internet zone
  • Require signed, vetted macros from trusted publishers
  • Open unknown documents in Protected View

Endpoint Hardening

  • Enable Attack Surface Reduction (ASR) rules
  • Block process injection and script abuse
  • Monitor startup entries for unauthorized modifications

Backup Policy

  • 3-2-1 principle: 3 copies, 2 different media types, 1 offsite/offline
  • Quarterly restore testing to verify backup integrity

Key Takeaways

  • Viruses replicate by inserting themselves into host programs, requiring user action or system processes to spread.
  • Multiple virus types exist, each with distinct infection strategies and evasion techniques.
  • Code insertion techniques range from crude overwriting to stealthy cavity injection.
  • Layered defense across people, process, and technology is essential for prevention, rapid detection, and recovery.
  • Regular patching, privilege restriction, and strong email controls are the most effective risk-reduction measures.
