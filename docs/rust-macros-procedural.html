Procedural Macros

Rust Proc Macros Metaprogramming

1. Overview of Procedural Macros

Procedural macros are Rust functions that receive a TokenStream and return a TokenStream. Unlike macro_rules!, they have access to the full Rust type system and can perform arbitrary computation at compile time.

There are three kinds:

  • Custom derive — #[derive(MyTrait)] generates a trait implementation.
  • Attribute macros — #[my_attr] transforms any item (functions, structs, etc.).
  • Function-like macros — my_macro!(...) like macro_rules! but with full Rust code.

1.1 Proc Macro Crate Setup

Proc macros must live in their own crate with proc-macro = true in Cargo.toml.

# my-derive/Cargo.toml
[package]
name = "my-derive"
version = "0.1.0"
edition = "2021"

[lib]
proc-macro = true

[dependencies]
syn   = { version = "2", features = ["full"] }
quote = "1"
proc-macro2 = "1"

The essential helper crates:

  • syn — parses Rust source into an AST (DeriveInput, ItemFn, etc.)
  • quote — quasi-quotes Rust code back into a TokenStream
  • proc-macro2 — a re-export of proc-macro types usable in non-proc-macro contexts (testing)

2. Custom Derive Macros

A derive macro receives the struct/enum definition and generates additional code — typically a trait implementation.

// my-derive/src/lib.rs
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Data, Fields};

/// Derives a `Describe` trait that prints the struct name and field names.
#[proc_macro_derive(Describe)]
pub fn derive_describe(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident; // the struct name

    // Extract field names (for named structs)
    let field_names: Vec<String> = match &input.data {
        Data::Struct(s) => match &s.fields {
            Fields::Named(f) => f.named.iter()
                .map(|f| f.ident.as_ref().unwrap().to_string())
                .collect(),
            _ => vec![],
        },
        _ => vec![],
    };

    let field_list = field_names.join(", ");

    // quote! generates the implementation as tokens
    let expanded = quote! {
        impl Describe for #name {
            fn describe(&self) -> String {
                format!(
                    "struct {} {{ {} }}",
                    stringify!(#name),
                    #field_list
                )
            }
        }
    };

    expanded.into()
}

Usage in the user's crate:

// user-crate/src/main.rs
// [dependencies] my-derive = { path = "../my-derive" }
use my_derive::Describe;

trait Describe {
    fn describe(&self) -> String;
}

#[derive(Describe)]
struct User {
    name: String,
    age: u32,
    email: String,
}

fn main() {
    let u = User {
        name: "Alice".to_string(),
        age: 30,
        email: "alice@example.com".to_string(),
    };
    println!("{}", u.describe());
    // struct User { name, age, email }
}

2.1 Derive with Helper Attributes

You can define helper attributes that annotate individual fields or variants:

// my-derive/src/lib.rs
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Data, Fields, Attribute, Meta, Lit};

#[proc_macro_derive(Builder, attributes(builder))]
pub fn derive_builder(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;
    let builder_name = quote::format_ident!("{}Builder", name);

    let fields = match &input.data {
        Data::Struct(s) => match &s.fields {
            Fields::Named(f) => f.named.iter().collect::<Vec<_>>(),
            _ => panic!("Builder only supports named fields"),
        },
        _ => panic!("Builder only supports structs"),
    };

    let field_names: Vec<_> = fields.iter().map(|f| &f.ident).collect();
    let field_types: Vec<_> = fields.iter().map(|f| &f.ty).collect();

    let expanded = quote! {
        struct #builder_name {
            #( #field_names: Option<#field_types>, )*
        }

        impl #builder_name {
            fn new() -> Self {
                Self { #( #field_names: None, )* }
            }

            #(
                fn #field_names(mut self, v: #field_types) -> Self {
                    self.#field_names = Some(v);
                    self
                }
            )*

            fn build(self) -> Result<#name, String> {
                Ok(#name {
                    #(
                        #field_names: self.#field_names.ok_or_else(||
                            format!("missing field: {}", stringify!(#field_names))
                        )?,
                    )*
                })
            }
        }

        impl #name {
            fn builder() -> #builder_name {
                #builder_name::new()
            }
        }
    };

    expanded.into()
}

Usage:

use my_derive::Builder;

#[derive(Builder, Debug)]
struct Config {
    host: String,
    port: u16,
    max_connections: u32,
}

fn main() {
    let config = Config::builder()
        .host("localhost".to_string())
        .port(8080)
        .max_connections(100)
        .build()
        .expect("invalid config");

    println!("{config:?}");
    // Config { host: "localhost", port: 8080, max_connections: 100 }
}

3. Attribute Macros

Attribute macros receive two token streams: the macro's arguments and the item it is applied to. They replace the entire item with their output.

// my-derive/src/lib.rs (same crate)
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, ItemFn};

/// Wraps a function to log entry and exit with timing.
/// Usage: #[timed]
#[proc_macro_attribute]
pub fn timed(args: TokenStream, input: TokenStream) -> TokenStream {
    let _ = args; // no arguments for this macro
    let func = parse_macro_input!(input as ItemFn);

    let func_name = &func.sig.ident;
    let func_vis  = &func.vis;
    let func_sig  = &func.sig;
    let func_body = &func.block;

    let expanded = quote! {
        #func_vis #func_sig {
            let __start = std::time::Instant::now();
            println!("[{}] entering", stringify!(#func_name));
            let __result = (|| #func_body)();
            println!("[{}] exited in {:?}", stringify!(#func_name), __start.elapsed());
            __result
        }
    };

    expanded.into()
}

Usage:

use my_derive::timed;

#[timed]
fn compute(n: u64) -> u64 {
    (1..=n).sum()
}

fn main() {
    let result = compute(1_000_000);
    println!("result: {result}");
    // [compute] entering
    // [compute] exited in 1.2ms
    // result: 500000500000
}

3.1 Attribute with Arguments

/// Retry a function up to N times on failure.
/// Usage: #[retry(3)]
#[proc_macro_attribute]
pub fn retry(args: TokenStream, input: TokenStream) -> TokenStream {
    let retries: usize = if args.is_empty() {
        3
    } else {
        // parse the literal argument
        let lit: syn::LitInt = syn::parse(args).expect("expected integer");
        lit.base10_parse().unwrap()
    };

    let func = parse_macro_input!(input as ItemFn);
    let func_name = &func.sig.ident;
    let func_vis  = &func.vis;
    let func_sig  = &func.sig;
    let func_body = &func.block;

    let expanded = quote! {
        #func_vis #func_sig {
            let mut __attempts = 0usize;
            loop {
                let __result: Result<_, _> = (|| { Ok(#func_body) })()
                    .and_then(|v: Result<_, _>| v);
                match __result {
                    Ok(v) => return Ok(v),
                    Err(e) if __attempts < #retries => {
                        __attempts += 1;
                        eprintln!("[{}] attempt {} failed: {:?}", stringify!(#func_name), __attempts, e);
                    }
                    Err(e) => return Err(e),
                }
            }
        }
    };

    expanded.into()
}

4. Function-Like Proc Macros

Function-like proc macros look like macro_rules! invocations but are implemented as Rust functions. They receive the tokens inside the parentheses.

// my-derive/src/lib.rs
use proc_macro::TokenStream;

/// Compile-time SQL validation (simplified illustration).
/// sql!("SELECT * FROM users WHERE id = ?")
#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {
    let sql_str = input.to_string();
    let sql_str = sql_str.trim_matches('"');

    // In a real implementation you'd parse and validate the SQL here
    if sql_str.to_uppercase().contains("DROP TABLE") {
        panic!("SQL validation failed: DROP TABLE is not allowed");
    }

    // Return a string constant (in practice you'd build a typed query)
    let quoted = format!("\"{}\"", sql_str);
    quoted.parse().unwrap()
}

5. Working with syn

syn parses Rust tokens into a rich AST. The most common entry points:

// Parse a struct/enum for derive macros
let input = parse_macro_input!(input as DeriveInput);
// input.ident    — struct/enum name
// input.generics — generic parameters
// input.data     — Data::Struct / Data::Enum / Data::Union

// Parse a function for attribute macros
let func = parse_macro_input!(input as ItemFn);
// func.sig.ident  — function name
// func.sig.inputs — parameters
// func.sig.output — return type
// func.block      — function body

// Parse arbitrary expressions
let expr = parse_macro_input!(input as syn::Expr);

// Iterate enum variants
if let Data::Enum(ref data) = input.data {
    for variant in &data.variants {
        let vname = &variant.ident;
        // variant.fields — Fields::Named / Fields::Unnamed / Fields::Unit
        // variant.discriminant — explicit discriminant (e.g., = 42)
    }
}

5.1 Handling Generics

Properly handling generic types is the trickiest part of proc macros:

use syn::{DeriveInput, Generics, GenericParam, WhereClause};
use quote::quote;

fn add_trait_bounds(mut generics: Generics, bound: syn::TypeParamBound) -> Generics {
    for param in &mut generics.params {
        if let GenericParam::Type(type_param) = param {
            type_param.bounds.push(bound.clone());
        }
    }
    generics
}

#[proc_macro_derive(MyClone)]
pub fn derive_my_clone(input: proc_macro::TokenStream) -> proc_macro::TokenStream {
    let input = syn::parse_macro_input!(input as DeriveInput);
    let name = &input.ident;

    // Add `: Clone` bound to every generic type parameter
    let generics = add_trait_bounds(
        input.generics.clone(),
        syn::parse_quote!(Clone),
    );

    let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

    let expanded = quote! {
        impl #impl_generics MyClone for #name #ty_generics #where_clause {
            fn my_clone(&self) -> Self {
                self.clone()
            }
        }
    };

    expanded.into()
}

6. Testing Procedural Macros

Use the trybuild crate to test that your macro compiles correct inputs and rejects invalid inputs with the right error messages.

# my-derive/Cargo.toml
[dev-dependencies]
trybuild = "1"

// my-derive/tests/integration.rs
#[test]
fn test_ui() {
    let t = trybuild::TestCases::new();
    t.pass("tests/ui/pass-*.rs");   // these must compile
    t.compile_fail("tests/ui/fail-*.rs"); // these must fail with expected errors
}

// tests/ui/pass-basic.rs
// use my_derive::Describe;

7.1 Using serde as a Reference

// serde is the gold standard for derive proc macros
// Cargo.toml: serde = { version = "1", features = ["derive"] }
//             serde_json = "1"
use serde::{Serialize, Deserialize};

#[derive(Debug, Serialize, Deserialize)]
struct Article {
    title: String,
    author: String,
    #[serde(rename = "published_at")]   // field-level helper attribute
    date: String,
    #[serde(default)]                    // use Default if field is missing
    views: u64,
    #[serde(skip_serializing_if = "Option::is_none")]
    subtitle: Option<String>,
}

fn main() -> serde_json::Result<()> {
    let json = r#"{
        "title": "Rust Macros",
        "author": "Paolo",
        "published_at": "2026-03-01"
    }"#;

    let article: Article = serde_json::from_str(json)?;
    println!("{article:?}");

    let back = serde_json::to_string_pretty(&article)?;
    println!("{back}");

    Ok(())
}

8. Tips & Best Practices

  • Emit good errors: use syn::Error::new_spanned(tokens, "message").to_compile_error() to point errors at the right location in user code.
  • Keep proc macro logic minimal: put the heavy logic in a separate library crate that the proc macro crate calls; this makes the logic testable without the compiler.
  • Use cargo expand to inspect generated code during development.
  • Avoid panicking: return a compile_error! token stream instead so the compiler reports a friendly error.
  • Mind compile times: proc macros run at compile time; keep them fast, avoid heavy computation.
// Emitting a compile error at the right span
fn check_input(input: &syn::DeriveInput) -> syn::Result<()> {
    match &input.data {
        syn::Data::Struct(_) => Ok(()),
        _ => Err(syn::Error::new_spanned(
            // The error will point at the enum/union keyword
            &input.ident,
            "MyDerive can only be applied to structs",
        )),
    }
}

#[proc_macro_derive(MyDerive)]
pub fn my_derive(input: proc_macro::TokenStream) -> proc_macro::TokenStream {
    let input = syn::parse_macro_input!(input as syn::DeriveInput);

    check_input(&input)
        .map(|_| {
            let name = &input.ident;
            quote::quote! {
                impl MyTrait for #name {}
            }.into()
        })
        .unwrap_or_else(|e| e.to_compile_error().into())
}
