IA-32 and Intel 64 ISA Assembly Language (Part 5)

Assembly IA-32 NASM

Part 5 introduces NASM (Netwide Assembler) — the portable x86 assembler that supports many object file formats including ELF, Mach-O, COFF/PE, OMF, and flat binaries. This installment covers program structure, directives, macros, calling conventions, and practical examples for both Linux and Windows environments.

Assembly Program Structure (NASM)

A typical NASM program is organized into sections that separate code from data:

  • section .data — initialized data and constants
  • section .bss — uninitialized storage (zeroed at load/start)
  • section .text — executable code

Minimal Linux x86-64 Example

; Minimal Linux x86-64 program: write(1,"Hello\n",6) then _exit(0)
; nasm -felf64 hello.asm -o hello.o
; ld hello.o -o hello

section .data
    msg     db  "Hello, world!", 10
    msg_len equ $ - msg

section .text
    global _start

_start:
    ; write(int fd, const void *buf, size_t count) -> syscall #1
    mov     rax, 1          ; SYS_write
    mov     rdi, 1          ; fd = stdout
    lea     rsi, [rel msg]  ; buffer
    mov     rdx, msg_len    ; length
    syscall

    ; exit(int status) -> syscall #60
    mov     rax, 60
    xor     rdi, rdi        ; status = 0
    syscall

Comments and Instruction Format

NASM comments start with a semicolon (;) and continue to end of line. The general instruction format is:

[label]    mnemonic    [operands]    [; comment]

Constants and Numbers

  • Decimal: 123
  • Hexadecimal: 0x7B or 7Bh
  • Binary: 1010b
  • Octal: 173o
  • Character: 'A'
  • String: "text"
  • Current location: $; difference: $ - label

Macros (NASM)

NASM macros are textual substitutions expanded before assembly. Parameters are referenced by %1, %2, and so on.

%macro print_string 2     ; %1=addr, %2=len  (Linux x86-64)
    mov rax, 1             ; SYS_write
    mov rdi, 1
    mov rsi, %1
    mov rdx, %2
    syscall
%endmacro

section .data
    hello   db "Hello, world!", 10
    hello_n equ $ - hello

section .text
    global _start
_start:
    print_string hello, hello_n
    mov rax, 60
    xor rdi, rdi
    syscall

Other Preprocessor Facilities

  • %define, %undef, %xdefine — symbolic substitution
  • %if, %elif, %else, %endif — conditional assembly
  • %assign, %rep, %endrep — numeric assignment and repetition
  • struc/endstruc — define data structures with offsets

Target Formats and Command Lines

Linux

Mode Assemble Link
x86-64 (ELF64) nasm -felf64 prog.asm -o prog.o ld prog.o -o prog or gcc prog.o -o prog
x86 (ELF32) nasm -felf32 prog.asm -o prog.o ld -m elf_i386 prog.o -o prog or gcc -m32 prog.o -o prog

Example: Call printf (ELF64, System V ABI)

; nasm -felf64 printf.asm -o printf.o
; gcc printf.o -o printf   # links libc
extern printf
section .data
    fmt db "sum=%d",10,0
section .text
    global main
main:
    push    rbp
    mov     rbp, rsp
    ; System V: RCX is temp; args in RDI, RSI, RDX, RCX, R8, R9
    mov     edi, fmt        ; 1st arg: format
    mov     esi, 40         ; 2nd arg: int
    xor     eax, eax        ; varargs: AL=0 XMM usage count
    call    printf
    xor     eax, eax
    leave
    ret

Windows

Mode Assemble Link
Win64 (COFF) nasm -fwin64 prog.asm -o prog.obj link /subsystem:console prog.obj (MSVC) or x86_64-w64-mingw32-gcc prog.obj -o prog.exe
Win32 (COFF) nasm -fwin32 prog.asm -o prog.obj link /subsystem:console prog.obj or i686-w64-mingw32-gcc prog.obj -o prog.exe

Windows x64 Calling Convention Notes

  • First four integer/pointer args in RCX, RDX, R8, R9 (callee may read a 32-byte shadow space at [rsp..rsp+32) reserved by caller)
  • Stack must be 16-byte aligned at call instruction
  • Floating args in XMM0–XMM3

Example: Call printf (Win64)

; nasm -fwin64 wprintf.asm -o wprintf.obj
; x86_64-w64-mingw32-gcc wprintf.obj -o wprintf.exe -lmsvcrt
extern printf
section .data
    fmt db "value=%d",13,10,0
section .text
    global main
main:
    ; Prologue and shadow space (32 bytes)
    sub     rsp, 40                 ; 32 shadow + 8 to keep 16-byte alignment
    lea     rcx, [rel fmt]          ; RCX = format
    mov     edx, 123                ; RDX = value
    xor     eax, eax                ; AL=0 for varargs
    call    printf
    xor     eax, eax
    add     rsp, 40
    ret

Bits Mode, Syntax, and Addressing

Commands bits 16 / bits 32 / bits 64 control instruction encoding. NASM infers mode from the -f target but explicit bits is useful inside mixed-mode sources. NASM uses Intel syntax: mnemonic dest, src. 64-bit code commonly uses RIP-relative addressing: lea rsi, [rel label].

Common Pitfalls and Best Practices

  • Stack alignment: Maintain 16-byte alignment before call on both SysV (Linux) and Win64; Windows additionally requires shadow space.
  • Preserved registers: Follow platform ABI. SysV preserves RBX, RBP, R12–R15; Win64 preserves RBX, RBP, RDI, RSI, R12–R15.
  • Global symbols: Export via global; import via extern.
  • Data vs code sections: Place strings and constants in .data; large zeroed buffers in .bss.
  • Relocations: Prefer [rel label] in x64 to generate RIP-relative references; avoid absolute addresses unless required.
  • System calls: Use OS ABI (syscall on Linux x86-64; int 0x80 legacy on x86; Windows uses user-mode APIs, not raw syscalls).

Instruction Set Notes for NASM Users

  • Use suffixes or size keywords when ambiguous: mov byte [rdi], 1, mov dword [rax], 0
  • Label scopes: NASM supports local labels prefixed with . under a non-local label
  • Structures: struc NAME ... endstruc defines field offsets; instantiate with istruc/iend or manual db/dw/...
  • Position-independent code: rely on rel/RIP and avoid absolute addresses to keep objects relocatable

Summary

NASM provides flexible x86 assembly with broad format support. Typical programs use .text, .data, and .bss; entry symbols and calling conventions vary by OS and bitness. Macros and directives keep code concise. For 64-bit development, follow platform ABI regarding argument registers, stack alignment, shadow space (Windows), and preserved registers.
