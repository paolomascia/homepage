Symmetric-Key Cryptography

Cryptography Symmetric Ciphers Stream Ciphers

Symmetric-key algorithms employ identical cryptographic keys for both encryption and decryption operations. These systems divide into two primary categories: stream ciphers and block ciphers.

Stream Ciphers

Stream ciphers process data sequentially, encrypting individual bits by combining plaintext bits with keystream bits through XOR operations. Stream ciphers encrypt the bit stream one bit at a time by combining each plaintext bit with a keystream bit.

Two stream cipher variants exist:

  • Synchronous: The keystream depends solely on the secret key.
  • Asynchronous: The keystream depends on previous ciphertext (feedback).

Stream ciphers offer advantages in constrained environments like mobile and embedded devices. Historical examples include A5/1 (GSM) and RC4 (early Wi-Fi), though RC4 is now considered insecure.

Block Ciphers

Block ciphers encrypt entire data blocks simultaneously, making each ciphertext bit dependent on multiple plaintext bits. DES uses 64-bit blocks; AES employs 128-bit blocks. Block ciphers dominate general-purpose applications including TLS and storage encryption.

Bitwise Operations

The fundamental encryption/decryption equation uses XOR:

y_i = x_i ⊕ s_i

Where y represents ciphertext, x represents plaintext, and s represents the keystream. XOR’s self-inverse property enables identical encryption and decryption processes.

Random Number Generators

Pseudorandom Number Generators (PRNG)

Deterministic sequences generated from seeds, exemplified by Linear Congruential Generators. Not suitable for cryptography without strengthening.

True Random Number Generators (TRNG)

TRNGs derive randomness from physical processes like thermal noise, providing non-reproducible outputs.

Cryptographically Secure PRNGs (CSPRNG)

CSPRNGs possess security properties preventing next-bit prediction and internal state recovery by polynomial-time algorithms.

One-Time Pad (OTP)

The sole unconditionally secure cipher requires three properties: true random keystreams, keystream secrecy, and single-use keystream bits. Given yi alone, xi is equiprobable (0 or 1) if si is truly random and used once. OTP impracticality stems from keystream length requirements—the keystream must be at least as long as the message itself.

Linear Feedback Shift Registers (LFSR)

LFSRs generate extended keystreams from brief secret seeds through shift-register operations with XOR feedback. Maximal LFSRs cycle through all non-zero states with primitive feedback polynomials.

// Shift register of degree m
// Feedback polynomial determines tap positions
// Output: one keystream bit per clock cycle
// Period: up to 2^m - 1 for maximal-length LFSRs

Trivium Stream Cipher

Trivium represents modern lightweight stream cipher design using three interlinked shift-register structures. Its specifications include:

  • 80-bit key
  • 80-bit initialization vector (IV)
  • Register lengths: 93, 84, and 111 bits

Operation involves initialization, a 1,152-cycle warm-up phase, and subsequent keystream generation. The IV need not be secret; its role is to ensure distinct keystreams even when the same key is reused.

// Trivium structure (conceptual)
// Three coupled nonlinear feedback shift registers
// Register A: 93 bits
// Register B: 84 bits
// Register C: 111 bits
// Total internal state: 288 bits

// Initialization:
//   Load key into Register A
//   Load IV into Register B
//   Set specific bits of Register C
//   Run 1152 warm-up clocks (output discarded)

// Keystream generation:
//   Each clock produces one output bit
//   Output = XOR of feedback from all three registers
