Strings: String, &str, OsString & Encoding

Rust Strings UTF-8

Rust has several string types, each serving a distinct purpose. The split between String and &str reflects ownership; the existence of OsString, CString, and PathBuf reflects the reality that not all strings in the world are UTF-8. Understanding why these types exist, when to use each, and how to convert between them is essential for writing Rust that interacts with the filesystem, the OS, or native libraries.

String vs &str

String is a heap-allocated, owned, growable sequence of UTF-8 bytes. It is the equivalent of std::string in C++ or StringBuilder in Java. &str is a borrowed reference to a UTF-8 string slice: a pointer and a length. It does not own the data; it is a view into memory owned by someone else — a String, a string literal baked into the binary, or any contiguous UTF-8 byte sequence.

The key rule: use &str in function parameters (it accepts both string literals and references to Strings via coercion), and use String when you need to own, build, or return string data.

The Ownership Model

Construction, coercion, cloning, and the distinction between owned and borrowed strings.

fn greet(name: &str) {
    println!("Hello, {name}!");
}

fn make_greeting(name: &str) -> String {
    format!("Hello, {name}!")
}

fn main() {
    // String literal: &'static str, embedded in binary
    let lit: &str = "hello world";

    // String: heap-allocated, owned
    let owned: String = String::from("hello world");
    let also: String = "hello".to_string();
    let concat: String = "hello".to_owned() + " world";

    // &String coerces to &str automatically (Deref coercion)
    greet("literal");     // &'static str
    greet(&owned);        // &String -> &str via Deref

    // &str to String: always requires allocation
    let s: String = lit.to_string();
    let s2: String = lit.to_owned();
    let s3: String = String::from(lit);

    // String to &str: zero-cost borrow
    let slice: &str = &owned;
    let partial: &str = &owned[0..5]; // "hello"

    println!("{}", make_greeting("Ferris"));
    println!("partial: {partial}");

    // Capacity and length (in bytes, not chars)
    let mut s = String::with_capacity(50);
    s.push_str("hello");
    s.push(' ');
    s.push_str("world");
    println!("len={}, cap={}", s.len(), s.capacity());
}

UTF-8 and Character Boundaries

Rust strings are guaranteed to be valid UTF-8. This means you cannot index a string with a byte offset that falls in the middle of a multibyte character — doing so would panic. The safe alternatives are: iterate over .chars() (Unicode scalar values), iterate over .bytes() (raw bytes), or use the .char_indices() iterator to get both byte offsets and chars simultaneously.

A char in Rust is a Unicode scalar value, encoded in 1-4 bytes as UTF-8. The length of a string in bytes (.len()) differs from its length in characters (.chars().count()) whenever multibyte characters are present. Never assume s.len() == s.chars().count().

Byte lengths vs character counts, safe slicing, char iteration, and handling multibyte characters correctly.

fn main() {
    let ascii   = "hello";
    let emoji   = "🦀 Ferris";
    let unicode = "日本語";

    // .len() is BYTE length, not character count
    println!("ascii   bytes={}, chars={}", ascii.len(), ascii.chars().count());   // 5, 5
    println!("emoji   bytes={}, chars={}", emoji.len(), emoji.chars().count());   // 11, 8
    println!("unicode bytes={}, chars={}", unicode.len(), unicode.chars().count()); // 9, 3

    // Safe iteration by char
    for (i, c) in emoji.char_indices() {
        println!("  byte_offset={i}, char={c:?}");
    }

    // Safe slicing: byte offsets must be on char boundaries
    let s = "héllo";
    // s[0..1] would panic: 'é' is 2 bytes
    // Safe: use char_indices to find boundaries
    let first_two: String = s.chars().take(2).collect();
    println!("first two chars: {first_two}"); // hé

    // Check if offset is a char boundary
    println!("is boundary at 1: {}", s.is_char_boundary(1)); // false
    println!("is boundary at 2: {}", s.is_char_boundary(2)); // true (after 'é')

    // Split at char boundary safely
    let (head, tail) = s.split_at(2); // 'h' + 'é' = 3 bytes... wait
    // Actually 'h'=1 byte, 'é'=2 bytes, so boundary at 3 for "hé"
    let (head, tail) = s.split_at(3);
    println!("head={head:?}, tail={tail:?}");

    // Bytes iterator
    let bytes: Vec<u8> = "hello".bytes().collect();
    println!("{bytes:?}"); // [104, 101, 108, 108, 111]
}

String Methods

The standard library provides a rich set of methods on &str and String. The most important patterns are searching, splitting, trimming, and replacing.

A comprehensive tour of the most commonly used string manipulation methods.

fn main() {
    let s = "  Hello, World! Hello, Rust!  ";

    // Trimming
    println!("{:?}", s.trim());          // "Hello, World! Hello, Rust!"
    println!("{:?}", s.trim_start());    // "Hello, World! Hello, Rust!  "
    println!("{:?}", s.trim_end());      // "  Hello, World! Hello, Rust!"

    // Searching
    println!("{}", s.contains("Rust"));          // true
    println!("{:?}", s.find("Hello"));           // Some(2)
    println!("{:?}", s.rfind("Hello"));          // Some(16)
    println!("{}", s.starts_with("  Hello"));    // true
    println!("{}", s.ends_with("!  "));          // true

    // Replacing
    let replaced = s.replace("Hello", "Hi");
    println!("{:?}", replaced);
    let once = s.replacen("Hello", "Hi", 1);
    println!("{:?}", once);

    // Splitting
    let words: Vec<&str> = "one,two,three".split(',').collect();
    println!("{words:?}"); // ["one", "two", "three"]

    let limited: Vec<&str> = "a:b:c:d".splitn(3, ':').collect();
    println!("{limited:?}"); // ["a", "b", "c:d"]

    // Lines
    let multiline = "line1\nline2\nline3";
    let lines: Vec<&str> = multiline.lines().collect();
    println!("{lines:?}");

    // Case
    println!("{}", "hello World".to_uppercase()); // HELLO WORLD
    println!("{}", "hello World".to_lowercase()); // hello world

    // Repeat and join
    println!("{}", "ab".repeat(3)); // ababab
    let parts = vec!["one", "two", "three"];
    println!("{}", parts.join(", ")); // one, two, three

    // Parsing from string
    let n: i32 = "42".parse().unwrap();
    let f: f64 = "3.14".parse().unwrap();
    let result: Result<i32, _> = "abc".parse::<i32>();
    println!("{n}, {f}, {result:?}");

    // Chars manipulation
    let capitalized: String = {
        let mut c = "hello".chars();
        match c.next() {
            None => String::new(),
            Some(first) => first.to_uppercase().to_string() + c.as_str(),
        }
    };
    println!("{capitalized}"); // Hello
}

Building Strings Efficiently

Concatenating strings with + is convenient but creates a new allocation for each operation. For building strings from many pieces, use String::with_capacity + push_str, or format! for complex cases. The write! macro from std::fmt::Write lets you write directly into a String with format strings, avoiding the temporary allocation that format! would create.

Comparing concatenation approaches: the + operator, format!, push_str, and write!.

use std::fmt::Write;

fn build_table(headers: &[&str], rows: &[Vec<String>]) -> String {
    let col_widths: Vec<usize> = headers.iter().enumerate().map(|(i, h)| {
        rows.iter()
            .map(|row| row.get(i).map(|s| s.len()).unwrap_or(0))
            .max()
            .unwrap_or(0)
            .max(h.len())
    }).collect();

    let mut out = String::with_capacity(256);
    let separator: String = col_widths.iter().map(|&w| "-".repeat(w + 2)).collect::<Vec<_>>().join("+");

    // write! into String -- no intermediate allocation
    writeln!(out, "+{separator}+").unwrap();
    write!(out, "|").unwrap();
    for (h, &w) in headers.iter().zip(col_widths.iter()) {
        write!(out, " {h:<w$} |").unwrap();
    }
    writeln!(out).unwrap();
    writeln!(out, "+{separator}+").unwrap();

    for row in rows {
        write!(out, "|").unwrap();
        for (cell, &w) in row.iter().zip(col_widths.iter()) {
            write!(out, " {cell:<w$} |").unwrap();
        }
        writeln!(out).unwrap();
    }
    writeln!(out, "+{separator}+").unwrap();
    out
}

fn main() {
    // + operator: allocates a new String at each +
    let s1 = "Hello".to_string();
    let s2 = ", ";
    let s3 = "World!";
    let result = s1 + s2 + s3; // s1 is moved, s2 and s3 are &str
    println!("{result}");

    // format!: most readable, but always allocates
    let name = "Rust";
    let greeting = format!("Hello, {name}! Version {}.{}", 1, 75);
    println!("{greeting}");

    // push_str: most efficient for incremental building
    let mut s = String::with_capacity(100);
    for i in 0..5 {
        if i > 0 { s.push_str(", "); }
        s.push_str(&i.to_string());
    }
    println!("{s}"); // 0, 1, 2, 3, 4

    // Table example
    let headers = ["Name", "Age", "City"];
    let rows = vec![
        vec!["Alice".to_string(), "30".to_string(), "Rome".to_string()],
        vec!["Bob".to_string(), "25".to_string(), "Milan".to_string()],
        vec!["Charlie".to_string(), "35".to_string(), "Florence".to_string()],
    ];
    print!("{}", build_table(&headers, &rows));
}

OsString, CString, and PathBuf

Not all strings that Rust programs encounter are valid UTF-8. The operating system and native libraries impose their own encoding conventions:

  • OsString / OsStr: the OS-native string type. On Unix, it is arbitrary bytes (typically UTF-8 but not guaranteed). On Windows, it is potentially ill-formed UTF-16. Used for file names, environment variables, process arguments, and OS paths. Converts losslessly from/to String only when valid UTF-8.
  • PathBuf / Path: a wrapper around OsString/OsStr that provides path-manipulation methods (join, extension, file_name, parent, etc.). Always use Path instead of string concatenation for paths.
  • CString / CStr: null-terminated byte strings for FFI with C libraries. The nul byte cannot appear in the string body; the terminating nul is managed by the type.

Working with PathBuf for filesystem operations, OsString for environment variables, and CString for FFI-ready strings.

use std::path::{Path, PathBuf};
use std::ffi::{OsString, CString, CStr};

fn describe_path(p: &Path) {
    println!("path:      {}", p.display());
    println!("file_name: {:?}", p.file_name());
    println!("extension: {:?}", p.extension());
    println!("parent:    {:?}", p.parent());
    println!("exists:    {}", p.exists());
}

fn main() {
    // PathBuf: owned path
    let mut path = PathBuf::from("/home/user");
    path.push("projects");        // /home/user/projects
    path.push("app");             // /home/user/projects/app
    path.set_extension("toml");   // /home/user/projects/app.toml

    describe_path(&path);

    // Path::new: borrowed path slice
    let p = Path::new("/etc/passwd");
    describe_path(p);

    // Join paths safely
    let base = Path::new("/var/log");
    let full = base.join("app").join("error.log");
    println!("full path: {}", full.display());

    // OsString: round-trip through env variable (may not be UTF-8)
    let key = OsString::from("PATH");
    if let Some(val) = std::env::var_os("PATH") {
        // to_string_lossy: converts to &str, replacing invalid UTF-8 with U+FFFD
        let s = val.to_string_lossy();
        let count = s.split(':').count();
        println!("PATH has {count} entries");
    }

    // CString: null-terminated, for C FFI
    let c_str = CString::new("hello from Rust").expect("no nul bytes allowed");
    // c_str.as_ptr() gives a *const c_char for passing to C functions

    // Converting: String -> CString -> &CStr -> &str
    let original = "hello";
    let c = CString::new(original).unwrap();
    let back: &str = c.to_str().unwrap();
    println!("round-trip: {back}");

    // Path to String (may fail if path is not valid UTF-8)
    let path_str: Option<&str> = full.to_str();
    println!("as str: {:?}", path_str);

    // Always-valid conversion with lossy
    let lossy = full.to_string_lossy();
    println!("lossy: {lossy}");
}

Cow<&str, str> — Clone on Write

Cow<'a, str> (Clone on Write) is a smart pointer that is either a borrowed &'a str or an owned String. It allows functions to return a string that is borrowed when no allocation is needed and owned when it is. This is perfect for sanitization or normalization functions: if the input is already in the desired form, return it borrowed; if it needs transformation, allocate and return owned.

A normalizer function that returns Cow<str> to avoid allocating when the input already conforms to the expected format.

use std::borrow::Cow;

fn normalize_whitespace(s: &str) -> Cow<str> {
    if s.contains("  ") || s.starts_with(' ') || s.ends_with(' ') {
        // Needs transformation: allocate
        Cow::Owned(s.split_whitespace().collect::<Vec<&str>>().join(" "))
    } else {
        // Already normalized: borrow
        Cow::Borrowed(s)
    }
}

fn ensure_uppercase_prefix(s: &str) -> Cow<str> {
    if s.starts_with(|c: char| c.is_uppercase()) {
        Cow::Borrowed(s)
    } else {
        let mut owned = s.to_owned();
        if let Some(first) = owned.get_mut(0..1) {
            first.make_ascii_uppercase();
        }
        Cow::Owned(owned)
    }
}

fn main() {
    let already_clean = "hello world";
    let needs_cleaning = "  hello   world  ";

    let r1 = normalize_whitespace(already_clean);
    let r2 = normalize_whitespace(needs_cleaning);

    // Both work as &str transparently
    println!("{:?}", r1); // Borrowed("hello world")   -- no allocation
    println!("{:?}", r2); // Owned("hello world")      -- allocated

    // Use as &str without caring which variant
    for s in [&r1, &r2] {
        println!("len={}", s.len());
    }

    // Cow in a Vec: heterogeneous borrowed/owned strings
    let items: Vec<Cow<str>> = vec![
        Cow::Borrowed("already fine"),
        Cow::Owned("built at runtime".to_string()),
        normalize_whitespace("  needs work  "),
    ];
    for item in &items { println!("{item}"); }
}
